# grid-template-columns 和 grid-template-rows

用于定义每一列的列宽和每一行的行高

# 语法

.container {
  grid-template-columns: <track-size> ... | <line-name> <track-size> ...;
  grid-template-rows: <track-size> ... | <line-name> <track-size> ...;
}
  • <track-size> 行高或者列宽的值。可以是长度值(px、em、rem 等)，百分比值，auto，以及 fr 单位(fraction 的缩写，意为"片段")
  • <line-name> 网格线的名称，以便后续引用(子项自行指定位置时，作为分割线的别名使用)

# 实例

.container {
  grid-template-columns: 1fr [split] auto 200px;
  grid-template-rows: 25% 1fr 2fr;
}

注意这里的 auto 的表现方式会有变化

# repeat()

用于简化重复的值，下面是一个九宫格的简单例子：

.container {
  grid-template-columns: repeat(3, 33.33%);
  grid-template-rows: repeat(3, 33.33%);
}
  • 第一个参数是要重复的次数
  • 第二个参数是要重复的值

# auto-fill 关键字

顾名思义，自动填充，也就是说在重复的值固定的情况下，重复的次数会自动计算

.container {
  grid-template-columns: repeat(auto-fill, 100px);
}

思考: 值必须是固定的吗？minmax 可以使用吗？貌似不能使用，只会按照 max 的值来计算。

# fr 关键字

fr 是单词 fraction 的缩写，表示分数，类似于 flex 中的 flex 属性

# 与固定值混用

会首先分配给固定值，然后剩下的空间按比例分配

.container {
  grid-template-columns: 200px 1fr 2fr;
}

假设总宽度是 500px，那么每 fr 的宽度就是 100px

# 与 auto 混用

分几种情况：

  • 当未设置 fr，则 auto 会自动拉伸
  • 当设置了 fr 并且 fr 的总和大于 1 的时候，auto 表现为内容的宽度。
  • 当设置了 fr 并且 fr 的总和小于 1 的使用，auto 的宽度为 容器宽度 - (容器宽度 - (auto 内容宽度)) * fr 总和

# minmax()

产生一个长度范围，表示长度就在这个范围之中。它接受两个参数，分别为最小值和最大值。

.container {
  grid-template-columns: 1fr 1fr minmax(100px, 1fr);
}

相当于第三列设置了最小宽度

# auto 关键字

这个值比较特别，除了与 fr 的混用表现形式多变外，还会触发很多的其他特性，后面会讲解
