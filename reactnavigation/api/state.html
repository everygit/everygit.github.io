# Navigation state 参考

导航状态是 React Navigation 存储应用程序的导航结构和历史记录的状态。 如果需要执行高级操作（如重置状态，提供自定义初始状态等），了解导航状态的结构很有用。

这是一个 JavaScript 对象，看起来像这样：

const state = {
  type: "stack",
  key: "stack-1",
  routeNames: ["Home", "Profile", "Settings"],
  routes: [
    { key: "home-1", name: "Home", params: { sortBy: "latest" } },
    { key: "settings-1", name: "Settings" },
  ],
  index: 1,
  stale: false,
};

每个导航状态对象中都存在少量属性：

type - 状态所属的导航器的类型，例如stack，tab，drawer。 key - 标识导航器的唯一键。 routeNames - 导航器中定义的屏幕名称。这是一个唯一的数组，其中包含每个屏幕的字符串。 routes - 在导航器中渲染的路由对象（屏幕）的列表。它还在堆栈导航器中表示历史记录。此数组中至少应存在一项。 index - 路由数组中焦点路由对象的索引。 history - 访问过的项目列表。这是一个可选属性，并非在所有导航器中都存在。例如，它仅出现在核心的选项卡式和抽屉式导航器中。历史记录数组中项目的形状可能会因导航器而异。此数组中至少应存在一项。 state - 除非 stale 属性显式设置为 false，否则导航状态被假定为陈旧的。这意味着状态对象需要 “rehydrated” (再融合？)。

routes 中的每个 route 对象都可以包含以下属性：

key - 屏幕的唯一键。自动创建或在导航到此屏幕时添加。 name - 屏幕名称。在导航器组件层次结构中定义。 params - 包含 params 的可选对象，该对象在浏览时定义，例如 navigate('Home', { sortBy: 'latest' })。 state - 一个可选对象，包含嵌套在此屏幕内的子导航器的导航状态。

例如，包含嵌套在其主屏幕内的选项卡导航器的堆栈导航器可能具有如下导航状态对象：

const state = {
  type: "stack",
  key: "stack-1",
  routeNames: ["Home", "Profile", "Settings"],
  routes: [
    {
      key: "home-1",
      name: "Home",
      state: {
        key: "tab-1",
        routeNames: ["Feed", "Library", "Favorites"],
        routes: [
          { key: "feed-1", name: "Feed", params: { sortBy: "latest" } },
          { key: "library-1", name: "Library" },
          { key: "favorites-1", name: "Favorites" },
        ],
        index: 0,
      },
    },
    { key: "settings-1", name: "Settings" },
  ],
  index: 1,
};

重要的是要注意，即使有嵌套的导航器，也不会在导航发生之前在 route 对象上添加 state 属性，因此不能保证该属性存在。

# 部分状态对象

较早前，在导航状态中提到了 stale 属性。过时的导航状态意味着状态对象在使用前需要 rehydrated 或 fixed 或 fixed up，例如添加丢失的键，删除无效的屏幕等。作为用户，您不必担心，除非将 stale 设置为 false，否则 React Navigation 将自动修复状态对象中的任何问题。如果要编写自定义路由器，可以使用 getRehydratedState 方法编写自定义 rehydration 逻辑以修复状态对象。

在执行诸如 reset，提供初始状态等操作时，此功能非常方便，因为您可以安全地从导航状态对象中省略许多属性，并依靠 React Navigation 为您添加这些属性，从而使您的代码更简单。例如，您只能提供一个没有任何键的 routes 数组，React Navigation 会自动添加使其正常工作所需的所有内容：

const state = {
  routes: [{ name: "Home" }, { name: "Profile" }],
};

rehydration 后，看起来会像这样：

const state = {
  type: "stack",
  key: "stack-1",
  routeNames: ["Home", "Profile", "Settings"],
  routes: [
    { key: "home-1", name: "Home" },
    { key: "settings-1", name: "Settings" },
  ],
  index: 1,
  stale: false,
};

在这里，React Navigation 填充了缺少的位，例如 key，routeNames，index 等。

还可以提供无效数据（例如不存在的屏幕），并且该数据将自动修复。 虽然不建议使用无效的状态对象编写代码，但是如果您进行状态持久性之类的操作（在更新后配置的屏幕可能已更改，则可能会导致问题，如果 React Navigation 无法自动地修复状态对象），则可能会非常有用。

如果要让 React Navigation 修复无效状态，则需要确保状态对象中没有stale: false。 状态为stale: false 的状态对象被认为是有效的状态对象，React Navigation 不会尝试对其进行修复。

当您在 initialState 中提供状态对象时，React Navigation 始终会认为它是一个过时的状态对象，这可以确保状态持久性之类的工作顺利进行而无需对状态对象进行额外的操作。
