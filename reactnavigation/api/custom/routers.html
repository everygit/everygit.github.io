# 自定义路由

路由器对象提供了各种帮助程序方法来处理状态和动作，简化程序以更新状态以及一些动作创建者。

路由器负责处理通过在导航对象上调用方法而分派的动作。如果路由器无法处理动作，则可以返回 null，这会将动作传播到其他路由器，直到处理完毕。

您可以通过构建具有以下功能的对象来制作自己的路由器：

  • type - 代表路由器类型的字符串，例如stack，tab，drawer等。
  • getInitialState - 返回导航器初始状态的函数。接收带有 routeNames 和 routeParamList 属性的 options 对象。
  • getRehydratedState - 从给定的部分状态重新补充整个导航状态的函数。接收带有 routeNames 和 routeParamList 属性的部分状态对象和选项对象。
  • getStateForRouteNamesChange - 获取当前状态和路由名称更新列表并返回新状态的函数。接收带有 routeNames 和 routeParamList 属性的状态对象和选项对象。
  • getStateForAction - 获取当前状态和操作以及带有 routeNames 和 routeParamList 属性的选项对象并返回新状态的函数。如果无法处理该操作，则应返回 null。
  • getStateForRouteFocus - 该函数获取路由的当前状态和键，并以该路由为中心返回新状态。
  • shouldActionChangeFocus - 确定操作是否也应更改父导航器中焦点的函数。某些操作（例如 NAVIGATE）可以更改父级中的焦点。
  • actionCreators - 包含动作创建者列表的可选对象，例如 push，pop 等。这些对象将用于向 navigation 对象添加辅助方法，以分派这些动作。

例子：

const router = {
  type: 'tab',

  getInitialState({ routeNames, routeParamList }) {
    const index =
      options.initialRouteName === undefined
        ? 0
        : routeNames.indexOf(options.initialRouteName);

    return {
      stale: false,
      type: 'tab',
      key: shortid(),
      index,
      routeNames,
      routes: routeNames.map(name => ({
        name,
        key: name,
        params: routeParamList[name],
      })),
    };
  },

  getRehydratedState(partialState, { routeNames, routeParamList }) {
    const state = partialState;

    if (state.stale === false) {
      return state as NavigationState;
    }

    const routes = state.routes
      .filter(route => routeNames.includes(route.name))
      .map(
        route =>
          ({
            ...route,
            key: route.key || `${route.name}-${shortid()}`,
            params:
              routeParamList[route.name] !== undefined
                ? {
                    ...routeParamList[route.name],
                    ...route.params,
                  }
                : route.params,
          } as Route<string>)
      );

    return {
      stale: false,
      type: 'tab',
      key: shortid(),
      index:
        typeof state.index === 'number' && state.index < routes.length
          ? state.index
          : 0,
      routeNames,
      routes,
    };
  },

  getStateForRouteNamesChange(state, { routeNames }) {
    const routes = state.routes.filter(route =>
      routeNames.includes(route.name)
    );

    return {
      ...state,
      routeNames,
      routes,
      index: Math.min(state.index, routes.length - 1),
    };
  },

  getStateForRouteFocus(state, key) {
    const index = state.routes.findIndex(r => r.key === key);

    if (index === -1 || index === state.index) {
      return state;
    }

    return { ...state, index };
  },

  getStateForAction(state, action) {
    switch (action.type) {
      case 'NAVIGATE': {
        const index = state.routes.findIndex(
          route => route.name === action.payload.name
        );

        if (index === -1) {
          return null;
        }

        return { ...state, index };
      }

      default:
        return BaseRouter.getStateForAction(state, action);
    }
  },

  shouldActionChangeFocus() {
    return false;
  },
};

const SimpleRouter = () => router;

export default SimpleRouter;

# 自定义路由器

您可以根据需要重用路由器并覆盖路由器功能，例如自定义如何处理现有操作，添加其他操作等。

有关如何在现有导航器中用自定义路由器覆盖路由器的详细信息，请参见自定义导航器。

# 自定义导航操作

假设您要添加自定义操作以清除历史记录：

import { TabRouter } from "@react-navigation/native";

const MyTabRouter = (options) => {
  const router = TabRouter(options);

  return {
    ...router,
    getStateForAction(state, action, options) {
      switch (action.type) {
        case "CLEAR_HISTORY":
          return {
            ...state,
            routeKeyHistory: [],
          };
        default:
          return router.getStateForAction(state, action, options);
      }
    },

    actionCreators: {
      ...router.actionCreators,
      clearHistory() {
        return { type: "CLEAR_HISTORY" };
      },
    },
  };
};

无需编写自定义路由器来处理自定义操作，而是可以传递一个函数来进行分派。 建议使用它更清洁，而不是取代路由器。

# 阻止导航操作

有时，根据您的路线，您可能希望阻止某些导航活动。 假设，如果 isEditing 为 true，则要防止推送新屏幕：

import { StackRouter } from "@react-navigation/native";

const MyStackRouter = (options) => {
  const router = StackRouter(options);

  return {
    ...router,
    getStateForAction(state, action, options) {
      const result = router.getStateForAction(state, action, options);

      if (
        result != null &&
        result.index > state.index &&
        state.routes[state.index].params?.isEditing
      ) {
        // Returning the current state means that the action has been handled, but we don't have a new state
        return state;
      }

      return result;
    },
  };
};

如果要防止返回，建议的方法是使用 beforeRemove 事件。
