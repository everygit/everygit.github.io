# 自定义导航

导航器允许您定义应用程序的导航结构。导航器还渲染可以配置的常见元素，例如标题和标签栏。

在后台，导航器是普通的 React 组件。

# 内置导航器

我们包括一些常用导航器，例如：

  • createStackNavigator - 一次渲染一个屏幕，并在屏幕之间提供过渡。打开新屏幕时，它将放置在堆栈顶部。
  • createDrawerNavigator - 提供一个默认情况下从屏幕左侧滑入的抽屉。
  • createBottomTabNavigator - 渲染选项卡栏，允许用户在多个屏幕之间切换。
  • createMaterialTopTabNavigator - 渲染选项卡视图，使用户可以使用滑动手势或选项卡栏在多个屏幕之间切换。
  • createMaterialBottomTabNavigator - 渲染选项卡视图，使用户可以使用滑动手势或选项卡栏在多个屏幕之间切换。

# 用于构建自定义导航器的 API

导航器将路由器和视图捆绑在一起，该视图获取导航状态并决定如何渲染它。我们导出一个 useNavigationBuilder 钩子来构建与其余 React Navigation 集成的自定义导航器。

# useNavigationBuilder

该挂钩允许组件挂钩到 React Navigation 中。它接受以下参数：

  • createRouter - 一种工厂方法，该方法返回路由器对象（例如 StackRouter，TabRouter）。
  • options - 挂钩和路由器的选项。导航器应在此处转发其 prop，以便用户可以提供用于配置导航器的 prop。默认情况下，接受以下选项：
  • children（必填） - children prop 应包含作为屏幕组件的路线配置。
  • screenOptions - screenOptions 属性应包含所有屏幕的默认选项。
  • initialRouteName - initialRouteName 属性确定屏幕以专注于初始渲染。该 prop 被转发到路由器。 如果此处传递了其他任何选项，它们将被转发到路由器。

挂钩返回具有以下属性的对象：

  • state - 导航器的导航状态。组件可以采用此状态并决定如何渲染它。
  • navigation - 导航对象，其中包含用于导航器的各种辅助方法，以操纵导航状态。这与屏幕的导航对象不同，它包括一些帮助程序，例如，emit 以向屏幕发送事件。
  • descriptors - 这是一个对象，其中包含每个路由的描述符，并且具有路由键作为其属性。路由的描述符可以通过描述符[route.key]访问。每个描述符包含以下属性：
    • navigation - 屏幕的导航 prop。您无需手动将其传递到屏幕。但是，如果我们在屏幕外也需要接收导航 prop 的组件（例如标题组件）渲染，这将很有用。
    • options - 一个 getter，它返回选项，例如屏幕 title（如果已指定）。
    • render - 可用于渲染实际屏幕的功能。调用描述符 descriptors[route.key].render() 将返回一个包含屏幕内容的 React 元素。使用此方法渲染屏幕非常重要，否则任何子导航器都无法正确连接到导航树。

例子：

import * as React from "react";
import { Text, TouchableOpacity, View } from "react-native";
import {
  NavigationHelpersContext,
  useNavigationBuilder,
  TabRouter,
  TabActions,
} from "@react-navigation/native";

function TabNavigator({
  initialRouteName,
  children,
  screenOptions,
  tabBarStyle,
  contentStyle,
}) {
  const { state, navigation, descriptors } = useNavigationBuilder(TabRouter, {
    children,
    screenOptions,
    initialRouteName,
  });

  return (
    <NavigationHelpersContext.Provider value={navigation}>
      <View style={[{ flexDirection: "row" }, tabBarStyle]}>
        {state.routes.map((route) => (
          <TouchableOpacity
            key={route.key}
            onPress={() => {
              const event = navigation.emit({
                type: "tabPress",
                target: route.key,
                canPreventDefault: true,
              });

              if (!event.defaultPrevented) {
                navigation.dispatch({
                  ...TabActions.jumpTo(route.name),
                  target: state.key,
                });
              }
            }}
            style={{ flex: 1 }}
          >
            <Text>{descriptors[route.key].options.title || route.name}</Text>
          </TouchableOpacity>
        ))}
      </View>
      <View style={[{ flex: 1 }, contentStyle]}>
        {descriptors[state.routes[state.index].key].render()}
      </View>
    </NavigationHelpersContext.Provider>
  );
}

导航器的 navigation 对象还具有发出方法，以将自定义事件 emit 到子屏幕。 用法如下所示：

navigation.emit({
  type: "transitionStart",
  data: { blurring: false },
  target: route.key,
});

data 在 event 对象的 data 属性下可用，即 event.data。

target 属性确定将接收事件的屏幕。 如果省略 target 属性，则事件将分派到导航器中的所有屏幕。

# createNavigatorFactory

此 createNavigatorFactory 函数用于创建将 Navigator 和 Screen 配对的函数。 自定义导航器需要在导出之前将导航器组件包装在 createNavigatorFactory 中。

例子：

import {
  useNavigationBuilder,
  createNavigatorFactory,
} from "@react-navigation/native";

// ...

export const createMyNavigator = createNavigatorFactory(TabNavigator);

然后可以像这样使用：

import { createMyNavigator } from "./myNavigator";

const My = createMyNavigator();

function App() {
  return (
    <My.Navigator>
      <My.Screen name="Home" component={HomeScreen} />
      <My.Screen name="Feed" component={FeedScreen} />
    </My.Navigator>
  );
}

# 类型检查导航器

要对导航器进行类型检查，我们需要提供 3 种类型：

  • 视图接受的 prop 类型
  • 支持的屏幕选项类型
  • 导航器发出的事件类型图

例如，要对我们的自定义标签导航器进行类型检查，我们可以执行以下操作：

import * as React from 'react';
import { StyleProp, ViewStyle } from 'react-native';
import {
  useNavigationBuilder,
  DefaultNavigatorOptions,
  TabRouter,
  TabActions,
  TabActionHelpers,
  TabRouterOptions,
  TabNavigationState,
  createNavigatorFactory,
} from '@react-navigation/native';

// Props accepted by the view
type TabNavigationConfig = {
  tabBarStyle: StyleProp<ViewStyle>;
  contentStyle: StyleProp<ViewStyle>;
};

// Supported screen options
type TabNavigationOptions = {
  title?: string;
};

// Map of event name and the type of data (in event.data)
//
// canPreventDefault: true adds the defaultPrevented property to the
// emitted events.
type TabNavigationEventMap = {
  tabPress: {
    data: { isAlreadyFocused: boolean }
    canPreventDefault: true
  };
};

// The props accepted by the component is a combination of 3 things
type Props = DefaultNavigatorOptions<TabNavigationOptions> &
  TabRouterOptions &
  TabNavigationConfig;

function TabNavigator({
  initialRouteName,
  children,
  screenOptions,
  tabBarStyle,
  contentStyle,
}: Props) {
  const { state, navigation, descriptors } = useNavigationBuilder<
    TabNavigationState<ParamListBase>,
    TabRouterOptions,
    TabActionHelpers<ParamListBase>,
    TabNavigationOptions,
    TabNavigationEventMap
  >(TabRouter, {
    children,
    screenOptions,
    initialRouteName,
  });

  return (
    <React.Fragment>
      <View style={[{ flexDirection: 'row' }, tabBarStyle]}>
        {state.routes.map(route => (
          <TouchableOpacity
            key={route.key}
            onPress={() => {
              const event = navigation.emit({
                type: 'tabPress',
                target: route.key,
                data: {
                  isAlreadyFocused: route.key === state.routes[state.index].key,
                },
              });

              if (!event.defaultPrevented) {
                navigation.dispatch({
                  ...TabActions.jumpTo(route.name),
                  target: state.key,
                });
              }
            }}
            style={{ flex: 1 }}
          >
            <Text>{descriptors[route.key].options.title || route.name}</Text>
          </TouchableOpacity>
        ))}
      </View>
      <View style={[{ flex: 1 }, contentStyle]}>
        {descriptors[state.routes[state.index].key].render()}
      </View>
    </React.Fragment>
  );
}

export default createNavigatorFactory<
  TabNavigationState<ParamListBase>,
  TabNavigationOptions,
  TabNavigationEventMap,
  typeof TabNavigator
>(TabNavigator);

# 扩展导航器

所有内置导航器均导出其视图，我们可以重复使用这些视图并在其之上构建其他功能。 例如，如果要重新构建底部的标签导航器，则需要以下代码：

import * as React from "react";
import {
  useNavigationBuilder,
  createNavigatorFactory,
  TabRouter,
} from "@react-navigation/native";
import { BottomTabView } from "@react-navigation/bottom-tabs";

function BottomTabNavigator({
  initialRouteName,
  backBehavior,
  children,
  screenOptions,
  ...rest
}) {
  const { state, descriptors, navigation } = useNavigationBuilder(TabRouter, {
    initialRouteName,
    backBehavior,
    children,
    screenOptions,
  });

  return (
    <BottomTabView
      {...rest}
      state={state}
      navigation={navigation}
      descriptors={descriptors}
    />
  );
}

export default createNavigatorFactory(BottomTabNavigator);

现在，我们可以对其进行自定义，以添加其他功能或更改行为。 例如，使用自定义路由器而不是默认的 TabRouter：

import MyRouter from "./MyRouter";

// ...

const { state, descriptors, navigation } = useNavigationBuilder(MyRouter, {
  initialRouteName,
  backBehavior,
  children,
  screenOptions,
});

// ...
