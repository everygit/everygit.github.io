# 社区开发导航器和库

本指南中列出的库可能尚未更新以使用最新版本的 React Navigation。请参阅库的文档以查看它支持哪个版本的 React Navigation。

# Navigators

# Fluid Transitions

Fluid Transitions 是一个库，它在使用 react-navigation 在屏幕之间导航期间提供共享元素过渡。

共享元素过渡是在导航过渡期间将一个屏幕中的元素可视化为另一屏幕中的相应元素的可视化。

该库实现了一个名为 FluidNavigator 的自定义导航器，使所有这一切和更多可能性成为可能。

# 链接

github.com/fram-x/FluidTransitions (opens new window)

# Libraries

# react-navigation-collapsible

react-navigation-collapsible 是一个库和一个高阶组件，它可以调整屏幕选项并使屏幕标题可折叠。

由于 react-navigation 的标题设计为 Animated 组件，因此可以通过将 Scrolled 或 FlatList 中的 Animated.Value 传递给标题来对标题进行动画处理。

# 链接

github.com/benevbright/react-navigation-collapsible (opens new window)

# react-native-screens

该项目旨在将本机导航容器组件公开给 React Native，并且从 2.14.0 版本开始，React Navigation 可以与其集成。使用 react-native-screens 带来了很多好处，例如在 iOS 上支持“可达性功能”，以及在两个平台上都改善了内存消耗。

# 链接

github.com/software-mansion/react-native-screens (opens new window)

# react-navigation-header-buttons

帮助您在导航栏中呈现按钮并处理样式，因此您不必这样做。它试图模仿本机导航栏按钮的外观，并尝试提供一个简单的界面供您与之交互。

# 链接

github.com/vonovak/react-navigation-header-buttons (opens new window)

# react-navigation-props-mapper

提供简单的 HOC，将反应导航道具直接映射到您的屏幕组件-即。您可以使用 const user = this.props.activeUser 代替 const user = this.props.route.params.activeUser。

# 链接

github.com/vonovak/react-navigation-props-mapper (opens new window)

# react-navigation-backhandler

通过基于组件的 API 的 React-Navigation 轻松处理 Android 后退按钮行为。
