# useLinking

useLinking 挂钩可让我们处理应用程序中的深层链接。 React Navigation 在内部使用它来实现深度链接支持。

您应该在 NavigationContainer 上使用 linking ，而不是直接使用此挂钩。 该文档适用于在添加 linking 之前已经在使用此挂钩的用户。

例子：

import * as React from "react";
import { ScrollView } from "react-native";
import { useLinking, NavigationContainer } from "@react-navigation/native";

export default function App() {
  const ref = React.useRef();

  const { getInitialState } = useLinking(ref, {
    prefixes: ["https://mychat.com", "mychat://"],
    config: {
      screens: {
        Chat: "feed/:sort",
      },
    },
  });

  const [isReady, setIsReady] = React.useState(false);
  const [initialState, setInitialState] = React.useState();

  React.useEffect(() => {
    getInitialState()
      .catch(() => {})
      .then((state) => {
        if (state !== undefined) {
          setInitialState(state);
        }

        setIsReady(true);
      });
  }, [getInitialState]);

  if (!isReady) {
    return null;
  }

  return (
    <NavigationContainer initialState={initialState} ref={ref}>
      {/* content */}
    </NavigationContainer>
  );
}

有关如何配置深层链接的完整指南，请参见深层链接指南。

# Options

# prefixes

要处理的 URL 前缀。 您可以提供多个前缀来支持自定义方案以及通用链接。

仅匹配这些前缀的 URL。 解析之前，将从 URL 中删除该前缀。

例子：

useLinking(ref, {
  prefixes: ["https://mychat.com", "mychat://"],
  config: {
    screens: {
      Chat: "feed/:sort",
    },
  },
});

仅 iOS 和 Android 支持此功能。

# config

配置以微调如何解析路径。 config 对象应代表应用程序中导航器的结构。

例如，如果我们在主屏幕内有目录屏幕，并希望它处理 item/:id 模式：

{
  screens: {
    Home: {
      screens: {
        Catalog: {
          path: 'item/:id',
          parse: {
            id: Number,
          },
        },
      },
    },
  }
}

解析选项可以是对象或字符串：

{
  screens: {
    Catalog: 'item/:id',
  }
}

指定字符串时，等效于提供 path 选项。

路径选项是一种与路径匹配的模式。 以 ： 开头的任何句段都将被识别为具有相同名称的参数。 例如 item/42 将被解析为 { name: 'item', params: { id: '42' } }。

initialRouteName 选项可确保在此处传递的路由名称将出现在导航器的状态中，例如 对于配置：

{
  screens: {
    Home: {
      initialRouteName: 'Feed',
      screens: {
        Catalog: {
          path: 'item/:id',
          parse: {
            id: Number,
          },
        },
        Feed: 'feed',
      },
    },
  }
}

和 URL：/item/42，状态将如下所示：

{
  routes: [
    {
      name: 'Home',
      state: {
        index: 1,
        routes: [
          {
            name: 'Feed'
          },
          {
            name: 'Catalog',
            params: { id: 42 },
          },
        ],
      },
    },
  ],
}

parse 选项控制如何解析参数。 在这里，您可以提供要解析的参数名称作为键，以及一个可以获取参数字符串值并返回已解析值的函数：

{
  screens: {
    Catalog: {
      path: 'item/:id',
      parse: {
        id: id => parseInt(id, 10),
      },
    },
  }
}

如果没有提供用于解析参数的自定义函数，它将被解析为字符串。

# enabled

可选的布尔值，用于启用或禁用链接集成。 默认为 true。

# getStateFromPath

您可以选择提供自己的实现，以覆盖 React Navigation 解析到状态对象的深层链接的方式。

例子：

useLinking(ref, {
  prefixes: ["https://mychat.com", "mychat://"],
  config: {
    screens: {
      Chat: "feed/:sort",
    },
  },
  getStateFromPath(path, config) {
    // Return a state object here
    // You can also reuse the default logic by importing `getStateFromPath` from `@react-navigation/native`
  },
});

# getPathFromState

您可以选择通过提供自己的实现来覆盖 React Navigation 序列化状态对象以进行链接的方式。 如果您指定了 getStateFromPath，这对于适当的 Web 支持是必需的。

例子：

useLinking(ref, {
  prefixes: ["https://mychat.com", "mychat://"],
  config: {
    screens: {
      Chat: "feed/:sort",
    },
  },
  getPathFromState(state, config) {
    // Return a path string here
    // You can also reuse the default logic by importing `getPathFromState` from `@react-navigation/native`
