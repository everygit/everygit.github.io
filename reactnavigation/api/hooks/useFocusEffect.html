# useFocusEffect

有时我们希望在聚焦屏幕时产生 side-effects。 side-effects 可能涉及添加事件侦听器，获取数据，更新文档标题等操作。虽然可以使用焦点和模糊事件来实现，但这并不是很符合人体工程学。

为了简化操作，该库导出了 useFocusEffect 钩子：

import { useFocusEffect } from "@react-navigation/native";

function Profile({ userId }) {
  const [user, setUser] = React.useState(null);

  useFocusEffect(
    React.useCallback(() => {
      const unsubscribe = API.subscribe(userId, (user) => setUser(user));

      return () => unsubscribe();
    }, [userId])
  );

  return <ProfileContent user={user} />;
}

注意：为避免过于频繁地运行效果，如示例所示，在将回调传递给 useFocusEffect 之前，将回调包装在 useCallback 中非常重要。

useFocusEffect 类似于 React 的 useEffect 钩子。唯一的区别是，它仅在屏幕当前处于焦点状态时才运行。

每当传递给 React.useCallback 的依赖项发生更改时，效果就会运行，即它将在初始渲染（如果屏幕已聚焦）上运行，并且在依赖项已更改的情况下在后续渲染上运行。如果您没有将效果包装在 React.useCallback 中，则在屏幕聚焦时，该效果将运行每个渲染。

当需要清除前一个效果时，即当依赖项发生变化并计划了新效果时，以及当屏幕卸载或模糊时，将运行清除功能。

# 运行异步效果

当运行异步效果（例如从服务器获取数据）时，确保您在清理函数中取消请求非常重要（类似于 React.useEffect）。如果您使用的 API 不提供取消机制，请确保忽略状态更新：

useFocusEffect(
  React.useCallback(() => {
    let isActive = true;

    const fetchUser = async () => {
      try {
        const user = await API.fetch({ userId });

        if (isActive) {
          setUser(user);
        }
      } catch (e) {
        // Handle error
      }
    };

    fetchUser();

    return () => {
      isActive = false;
    };
  }, [userId])
);

如果您不忽略结果，则可能会由于 API 调用中的竞争条件而导致数据不一致。

# 延迟效果直到过渡完成

屏幕聚焦后，useFocusEffect 挂钩立即运行效果。 这通常意味着，如果有用于屏幕切换的动画，则可能尚未完成。

React Navigation 在本机线程中运行其动画，因此在很多情况下这不是问题。 但是，如果该效果更新了 UI 或渲染了一些昂贵的东西，则可能会影响动画性能。 在这种情况下，我们可以使用 InteractionManager 将工作推迟到动画或手势完成之前：

useFocusEffect(
  React.useCallback(() => {
    const task = InteractionManager.runAfterInteractions(() => {
      // Expensive task
    });

    return () => task.cancel();
  }, [])
);

# useFocusEffect 与为 focus 事件添加侦听器有何不同？

当屏幕成为焦点时，将触发焦点事件。 由于这是一个事件，因此当您订阅该事件时，如果屏幕已经聚焦，则不会调用您的侦听器。 当屏幕未聚焦时，这也没有提供执行清除功能的方法。 您可以订阅 blur 事件并手动处理它，但它可能会变得凌乱。 除了这些事件之外，您通常还需要处理 componentDidMount 和 componentWillUnmount，这使它更加复杂。

useFocusEffect 允许您对焦点运行效果，并在屏幕未对焦时清理效果。 它还可以处理卸载时的清理。 当依赖项更改时，它将重新运行效果，因此您无需担心侦听器中的陈旧值。

# 与类组件一起使用

您可以制作一个效果组件，并在类组件中使用它：

function FetchUserData({ userId, onUpdate }) {
  useFocusEffect(
    React.useCallback(() => {
      const unsubscribe = API.subscribe(userId, onUpdate);

      return () => unsubscribe();
    }, [userId, onUpdate])
  );

  return null;
}

// ...

class Profile extends React.Component {
  _handleUpdate = (user) => {
    // Do something with user object
  };

  render() {
    return (
      <>
        <FetchUserData
          userId={this.props.userId}
          onUpdate={this._handleUpdate}
        />
        {/* rest of your code */}
      </>
    );
  }
}
