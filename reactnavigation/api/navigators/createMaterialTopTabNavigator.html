# createMaterialTopTabNavigator

屏幕顶部的以 material-design 为主题的标签栏，可让您通过点击标签或水平滑动来在不同的路由之间切换。 过渡默认为动画。 每个路由的屏幕组件都将立即安装。

这包含了 react-native-tab-view。

要使用此导航器，请确保您具有 @react-navigation/native 及其依赖项（遵循本指南），然后安装 @react-navigation/material-top-tabs：

npm install @react-navigation/material-top-tabs react-native-tab-view@^2.16.0

# API 定义

要使用此标签导航器，请从 @react-navigation/material-top-tabs 导入：

import { createMaterialTopTabNavigator } from "@react-navigation/material-top-tabs";

const Tab = createMaterialTopTabNavigator();

function MyTabs() {
  return (
    <Tab.Navigator>
      <Tab.Screen name="Home" component={HomeScreen} />
      <Tab.Screen name="Settings" component={SettingsScreen} />
    </Tab.Navigator>
  );
}

有关完整的使用指南，请访问 选项卡导航

# Props

Tab.Navigator 组件接受以下 props：

# initialRouteName

导航器首次加载时要渲染的路由的名称。

# screenOptions

用于导航器中的屏幕的默认选项。

# backBehavior

后退按钮处理的行为。

  • initialRoute 返回初始标签
  • order 返回到上一个选项卡（按它们在选项卡栏中显示的顺序）
  • history 以返回到上次访问的标签页
  • none 不处理后退按钮

# tabBarPosition

标签栏在标签视图中的位置。可能的值为 'top' 和 'bottom'。默认为 'top'。

# lazy

指示是否延迟渲染场景的布尔值。设置为 true 时，屏幕进入视口时将进行渲染。默认情况下，将渲染所有场景以提供更流畅的滑动体验。但是您可能希望将屏幕的渲染推迟到视口之外，直到用户看到它们为止。要启用延迟渲染，请将 lazy 设置为 true。

启用延迟后，延迟加载的屏幕进入视口时通常会花费一些时间进行渲染。您可以使用 lazyPlaceholder 属性来自定义用户在这短时间内看到的内容。

# lazyPreloadDistance

启用 lazy 后，您可以指定使用此 prop 预先预载多少个相邻路由。该值默认为 0，这意味着 lazy 页面进入视口时将被加载。

# lazyPlaceholder

返回一个 React 元素以渲染尚未渲染的路由的函数。接收包含路由作为参数的对象。还需要启用lazy prop。

通常只在一秒钟内显示此视图。保持轻巧。

默认情况下，这将渲染 null。

# removeClippedSubviews

布尔值，指示是否从本机视图层次结构中删除不可见的视图（例如未聚焦的屏幕）以提高内存使用率。默认为 false。

注意：请勿在有问题且不会重新显示视图的 iOS 上启用此功能。

# keyboardDismissMode

指示键盘在响应拖动手势时是否被关闭的字符串。可能的值是:

  • 'auto'（默认）：index 更改时关闭键盘。
  • 'on-drag'：拖动开始时，键盘被关闭。
  • 'none'：拖动不会关闭键盘。

# swipeEnabled

布尔值，指示是否启用滑动手势。默认情况下，滑动手势是启用的。传递 false 将禁用滑动手势，但是用户仍然可以通过按选项卡栏来切换选项卡。

# swipeVelocityImpact

确定在滑动时计算下一个位置时速度的相关性。默认值为 0.2。

# TimingConfig

点击选项卡时发生的定时动画的配置对象。支持的属性是：

  • duration（number）

# springConfig

刷卡后发生的弹簧动画的配置对象。支持的属性是：

  • damping （number）
  • mass （number）
  • stiffness （number）
  • restSpeedThreshold（number）
  • restDisplacementThreshold （number）

# springVelocityScale

用于确定手势速度对计算弹簧动画的初始速度有多有意义的数字。预设为 0。

# initialLayout

包含屏幕初始高度和宽度的对象。通过此操作将提高初始渲染性能。对于大多数应用程序，这是一个很好的默认设置：

{ width: Dimensions.get('window').width }}

# position

动画值（来自 react-native-reanimated）以侦听位置更新。 传递的位置值将与选项卡的当前位置保持同步。 对于在选项卡视图之外访问动画值很有用。

# sceneContainerStyle

应用于环绕每个屏幕的视图的样式。 您可以传递此值来覆盖一些默认样式，例如溢出裁剪。

# style

应用于标签视图容器的样式。

# gestureHandlerProps

带有道具的对象将传递给基础 PanGestureHandler (opens new window)。 例如：

gestureHandlerProps={{
  maxPointers: 1,
  waitFor: [someRef]
}}

# pager

返回一个 React 元素用作 pager 的函数。 pager 处理滑动手势和页面切换。 默认情况下，我们使用 react-native-gesture-handler 处理手势。 您可以将 pager 切换为其他实现，以自定义体验。

例如，要使用本机 ViewPager 支持的 pager，可以使用 react-native-tab-view-viewpager-adapter：

import ViewPagerAdapter from 'react-native-tab-view-viewpager-adapter';

// ...

<Tab.Navigator pager={props => <ViewPagerAdapter {...props} />}>
  {...}
</Tab.Navigator>

# tabBar

返回 React 元素以显示为选项卡栏的函数。

例子：

import { View, TouchableOpacity } from 'react-native';
import Animated from 'react-native-reanimated';

function MyTabBar({ state, descriptors, navigation, position }) {
  return (
    <View style={{ flexDirection: 'row' }}>
      {state.routes.map((route, index) => {
        const { options } = descriptors[route.key];
        const label =
          options.tabBarLabel !== undefined
            ? options.tabBarLabel
            : options.title !== undefined
            ? options.title
            : route.name;

        const isFocused = state.index === index;

        const onPress = () => {
          const event = navigation.emit({
            type: 'tabPress',
            target: route.key,
            canPreventDefault: true,
          });

          if (!isFocused && !event.defaultPrevented) {
            navigation.navigate(route.name);
          }
        };

        const onLongPress = () => {
          navigation.emit({
            type: 'tabLongPress',
            target: route.key,
          });
        };

        const inputRange = state.routes.map((_, i) => i);
        const opacity = Animated.interpolate(position, {
          inputRange,
          outputRange: inputRange.map(i => (i === index ? 1 : 0)),
        });

        return (
          <TouchableOpacity
            accessibilityRole="button"
            accessibilityState={isFocused ? { selected: true } : {}}
            accessibilityLabel={options.tabBarAccessibilityLabel}
            testID={options.tabBarTestID}
            onPress={onPress}
            onLongPress={onLongPress}
            style={{ flex: 1 }}
          >
            <Animated.Text style={{ opacity }}>
              {label}
            </Animated.Text>
          </TouchableOpacity>
        );
      })}
    </View>
  );
}

// ...

<Tab.Navigator tabBar={props => <MyTabBar {...props} />}>
  {...}
</Tab.Navigator>

本示例将渲染带有标签的基本选项卡栏。

请注意，您不能使用 tabBar 内的 useNavigation 挂钩，因为 useNavigation 仅在屏幕内部可用。 您为 tabBar 获得了一个navigation prop，可以代替使用：

function MyTabBar({ navigation }) {
  return (
    <Button
      title="Go somewhere"
      onPress={() => {
        // Navigate using the `navigation` prop that you received
        navigation.navigate("SomeScreen");
      }}
    />
  );
}

# tabBarOptions

包含选项卡栏组件的道具的对象。它可以包含以下属性：

  • activeTintColor - 活动选项卡的标签和图标颜色。
  • inactiveTintColor - 非活动选项卡的标签和图标颜色。
  • showIcon - 是否显示选项卡图标，默认为 false。
  • showLabel - 是否显示标签标签，默认为 true。
  • pressColor - 材质波纹的颜色（仅适用于 Android> = 5.0）。
  • pressOpacity - 按下的标签的不透明度（仅适用于 iOS 和 Android <5.0）。
  • scrollEnabled - 是否启用可滚动的选项卡。
  • tabStyle - 选项卡的样式对象。
  • indicatorStyle - 选项卡指示器的样式对象（选项卡底部的行）。
  • labelStyle - 选项卡标签的样式对象。在此处指定颜色将覆盖在 activeTintColor 和 inactiveTintColor 中为标签指定的颜色。
  • iconStyle - 选项卡图标的样式对象。
  • style - 标签栏的样式对象。
  • allowFontScaling - 标签字体是否应缩放以符合“文本大小”辅助功能设置，默认为 true。
  • renderIndicator - 函数，该对象采用当前路由的对象并返回自定义的 React 元素以用作标签指示符。

例子：

<Tab.Navigator
  tabBarOptions={{
    labelStyle: { fontSize: 12 },
    tabStyle: { width: 100 },
    style: { backgroundColor: "powderblue" },
  }}
>
  {/* ... */}
</Tab.Navigator>

# Options

以下选项可用于配置导航器中的屏幕：

# title

通用标题，可用作 headerTitle 和 tabBarLabel 的后备。

# tabBarIcon

给定 { focused: boolean, color: string } 的函数返回一个 React.Node，以显示在选项卡栏中。

# tabBarLabel

显示在选项卡栏中的选项卡的标题字符串或给定{ focused: boolean, color: string }的函数返回一个 React.Node，以显示在选项卡栏中。未定义时，将使用场景 title。要隐藏，请参阅上一节中的 tabBarOptions.showLabel。

# tabBarAccessibilityLabel

选项卡按钮的辅助功能标签。当用户点击选项卡时，将由屏幕阅读器读取。如果您没有该标签的标签，建议您进行设置。

# tabBarTestID

在测试中找到此选项卡按钮的 ID。

# Events

导航器可以在某些操作上发出事件。支持的事件有：

# tabPress

当用户在选项卡栏中按当前屏幕的选项卡按钮时，将触发此事件。默认情况下，按 Tab 可做几件事：

  • 如果选项卡未聚焦，则按 Tab 将使该选项卡聚焦
  • 如果选项卡已经聚焦：
    • 如果选项卡的屏幕渲染滚动视图，则可以使用 useScrollToTop 将其滚动到顶部
    • 如果选项卡的屏幕渲染堆栈导航器，则会在堆栈上执行 popToTop 操作

为了防止出现默认行为，您可以调用 event.preventDefault：

React.useEffect(() => {
  const unsubscribe = navigation.addListener("tabPress", (e) => {
    // Prevent default behavior
    e.preventDefault();

    // Do something manually
    // ...
  });

  return unsubscribe;
}, [navigation]);

# tabLongPress

当用户长时间按下选项卡栏中当前屏幕的选项卡按钮时，将触发此事件。

例子：

React.useEffect(() => {
  const unsubscribe = navigation.addListener("tabLongPress", (e) => {
    // Do something
  });

  return unsubscribe;
}, [navigation]);

# Helpers

标签导航器将以下方法添加到导航道具：

# jumpTo

导航到选项卡导航器中的现有屏幕。 该方法接受以下参数：

name - string - 跳转到的路由名称。 params - object - 合并到目标路由的屏幕参数（通过 route.params 在推送的屏幕中找到）。

navigation.jumpTo("Profile", { name: "Michaś" });

# Example

import { createMaterialTopTabNavigator } from "@react-navigation/material-top-tabs";

const Tab = createMaterialTopTabNavigator();

function MyTabs() {
  return (
    <Tab.Navigator
      initialRouteName="Feed"
      tabBarOptions={{
        activeTintColor: "#e91e63",
        labelStyle: { fontSize: 12 },
        style: { backgroundColor: "powderblue" },
      }}
    >
      <Tab.Screen
        name="Feed"
        component={Feed}
        options={{ tabBarLabel: "Home" }}
      />
      <Tab.Screen
        name="Notifications"
        component={Notifications}
        options={{ tabBarLabel: "Updates" }}
      />
      <Tab.Screen
        name="Profile"
        component={Profile}
        options={{ tabBarLabel: "Profile" }}
      />
    </Tab.Navigator>
  );
}
