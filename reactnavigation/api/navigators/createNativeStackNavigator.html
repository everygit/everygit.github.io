# createNativeStackNavigator

为您的应用提供一种在每个新屏幕都放在堆栈顶部的屏幕之间过渡的方式。

该导航器使用 iOS 上的本机 API UINavigationController 和 Android 上的 Fragment，以便使用 createNativeStackNavigator 构建的导航的行为与在这些 API 上本机构建的应用程序完全相同，并且具有相同的性能特征。

折衷方案是 createNativeStackNavigator 的可定制性不高，因此有时您可能想要使用 createStackNavigator 来获得所需的外观或行为。

该导航器当前不支持 web。 改用 createStackNavigator 进行应用程序中的 Web 导航。

要使用此导航器，请确保您具有 @react-navigation/native 及其依赖项（遵循本指南）。

# API 定义

💡 如果在使用 createNativeStackNavigator 时遇到任何错误，请在 react-native-screens 而不是 react-navigation 存储库上打开问题。

要使用此导航器，请从 react-native-screens 导入 enableScreens 并调用它，然后从 react-native-screens/native-stack 导入 createNativeStackNavigator：

import { enableScreens } from "react-native-screens";
import { createNativeStackNavigator } from "react-native-screens/native-stack";

enableScreens();
const Stack = createNativeStackNavigator();

function MyStack() {
  return (
    <Stack.Navigator>
      <Stack.Screen name="Home" component={Home} />
      <Stack.Screen name="Notifications" component={Notifications} />
      <Stack.Screen name="Profile" component={Profile} />
      <Stack.Screen name="Settings" component={Settings} />
    </Stack.Navigator>
  );
}

有关最新的完整 API 参考，请参阅 README，以获取 react-native-screens/native-stack (opens new window)。
