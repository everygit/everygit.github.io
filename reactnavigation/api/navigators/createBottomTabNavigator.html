# createBottomTabNavigator

屏幕底部的简单 tab bar 可让您在不同的路由之间切换。 路由被延迟初始化 - 直到它们首先被聚焦时才安装它们的屏幕组件。

要使用此导航器，请确保您具有 @react-navigation/native 及其依赖项（遵循本指南），然后安装 @react-navigation/bottom-tabs：

npm install @react-navigation/bottom-tabs

# API 定义

要使用此标签导航器，请从 @react-navigation/bottom-tabs 导入它：

import { createBottomTabNavigator } from "@react-navigation/bottom-tabs";

const Tab = createBottomTabNavigator();

function MyTabs() {
  return (
    <Tab.Navigator>
      <Tab.Screen name="Home" component={HomeScreen} />
      <Tab.Screen name="Settings" component={SettingsScreen} />
    </Tab.Navigator>
  );
}

有关完整的使用指南，请访问 选项卡导航

# Props

Tab.Navigator 组件接受以下 prop：

# initialRouteName

导航器首次加载时要渲染的路由的名称。

# screenOptions

用于导航器中的屏幕的默认选项。

# backBehavior

后退在导航器中的行为方式，例如 当调用 goBack 或按下设备后退按钮时。

  • initialRoute 返回到初始选项卡
  • order 返回到上一个选项卡（按照它们在选项卡栏中显示的顺序）
  • history 返回上次访问的选项卡
  • none 不处理后退按钮

# lazy

默认为 true。 如果为 false，则立即显示所有选项卡。 如果为 true，则仅在首次使选项卡处于活动状态时才渲染这些选项卡。 注意：选项卡不会在后续访问时重新渲染。

# detachInactiveScreens

布尔值，用于指示是否应将非活动屏幕从视图层次结构中分离出来以节省内存。 确保从 react-native-screens 调用 enableScreens 使其起作用。 默认为 true。

# sceneContainerStyle

包装屏幕内容的组件的样式对象。

# tabBar

返回 React 元素以显示为选项卡栏的函数。

例子：

import { View, Text, TouchableOpacity } from 'react-native';

function MyTabBar({ state, descriptors, navigation }) {
  const focusedOptions = descriptors[state.routes[state.index].key].options;

  if (focusedOptions.tabBarVisible === false) {
    return null;
  }

  return (
    <View style={{ flexDirection: 'row' }}>
      {state.routes.map((route, index) => {
        const { options } = descriptors[route.key];
        const label =
          options.tabBarLabel !== undefined
            ? options.tabBarLabel
            : options.title !== undefined
            ? options.title
            : route.name;

        const isFocused = state.index === index;

        const onPress = () => {
          const event = navigation.emit({
            type: 'tabPress',
            target: route.key,
            canPreventDefault: true,
          });

          if (!isFocused && !event.defaultPrevented) {
            navigation.navigate(route.name);
          }
        };

        const onLongPress = () => {
          navigation.emit({
            type: 'tabLongPress',
            target: route.key,
          });
        };

        return (
          <TouchableOpacity
            accessibilityRole="button"
            accessibilityState={isFocused ? { selected: true } : {}}
            accessibilityLabel={options.tabBarAccessibilityLabel}
            testID={options.tabBarTestID}
            onPress={onPress}
            onLongPress={onLongPress}
            style={{ flex: 1 }}
          >
            <Text style={{ color: isFocused ? '#673ab7' : '#222' }}>
              {label}
            </Text>
          </TouchableOpacity>
        );
      })}
    </View>
  );
}

// ...

<Tab.Navigator tabBar={props => <MyTabBar {...props} />}>
  {...}
</Tab.Navigator>

本示例将渲染带有标签的基本选项卡栏。

请注意，您不能使用 tabBar 内的 useNavigation hook，因为 useNavigation 仅在屏幕内部可用。 您为 tabBar 获得了一个 navigation prop，可以代替使用：

function MyTabBar({ navigation }) {
  return (
    <Button
      title="Go somewhere"
      onPress={() => {
        // Navigate using the `navigation` prop that you received
        navigation.navigate("SomeScreen");
      }}
    />
  );
}

# tabBarOptions

包含默认选项卡栏组件的 props 的对象。 如果您使用的是自定义 tab bar，这些将作为 prop 传递到 tab bar，您可以对其进行处理。

它可以包含以下属性：

# activeTintColor

活动选项卡项目的标签和图标颜色。

# inactiveTintColor

非活动标签项的标签和图标颜色。

# activeBackgroundColor

活动选项卡项目的背景色。

# inactiveBackgroundColor

非活动选项卡项目的背景颜色。

# tabStyle

选项卡项目的样式对象。

# showLabel

是否显示标签标签，默认为 true。

# labelStyle

选项卡标签文本的样式对象。

# labelPosition

标签是在图标下方还是在图标旁边渲染。 可能的值为：

  • below-icon
  • beside-icon

默认情况下，在 vertical 方向（纵向模式）下，标签显示在图标下方，在 horizontal 方向（横向模式）下，标签显示在图标旁边。

# adaptive

标签图标和标签对齐方式是否应根据屏幕尺寸而改变？默认为 true。如果为 false，则选项卡图标和标签始终始终垂直对齐（labelPosition: 'below-icon'）。如果为 true，则标签图标和标签在平板电脑上水平对齐（labelPosition: 'beside-icon'）。

# allowFontScaling

标签字体是否应缩放以符合“文本大小”辅助功能设置，默认为 true。

# keyboardHidesTabBar

键盘打开时选项卡栏是否隐藏。默认为 false。

# safeAreaInsets

屏幕的安全区域插图。这用于避免使用诸如缺口和系统导航栏之类的元素。默认情况下，将自动检测设备的安全区域插图。您可以使用此选项覆盖行为。

接受包含以下可选属性的对象：

  • top - number - 顶部插图的值，例如包含状态栏和刻痕的区域。
  • right - number - 左插图的值。
  • bottom - number - 底部插图的值，例如底部的区域导航栏。
  • left - number - 右插图的值。
# style

tab bar 的样式对象。您可以在此处配置样式，例如背景色。

要在选项卡栏下显示屏幕，可以将 position 样式设置为 absolute：

style: {
  position: "absolute";
}

如果您有一个绝对定位的 tab bar，则可能还需要在内容中添加一个底边距。 React Navigation 不会自动执行。

要获取底部 tab bar 的高度，可以将 BottomTabBarHeightContext 与 React 的 Context API 一起使用，或者将 useBottomTabBarHeight 用作：

import { BottomTabBarHeightContext } from '@react-navigation/bottom-tabs';

// ...

<BottomTabBarHeightContext.Consumer>
  {tabBarHeight => (
    /* render something */
  )}
</BottomTabBarHeightContext.Consumer>

或

import { useBottomTabBarHeight } from "@react-navigation/bottom-tabs";

// ...

const tabBarHeight = useBottomTabBarHeight();

# Options

以下选项可用于配置导航器中的屏幕：

# title

通用标题，可用作 headerTitle 和 tabBarLabel 的后备。

# tabBarVisible

true 或 false 显示或隐藏选项卡栏，如果未设置，则默认为 true。

注意：隐藏选项卡栏可能会导致毛刺和抖动行为。我们建议将选项卡导航器放在堆栈导航器内部。

# tabBarIcon

给定 { focused: boolean, color: string, size: number } 的函数返回一个 React.Node，以显示在选项卡栏中。

# tabBarLabel

显示在选项卡栏中的选项卡的标题字符串或给定 { focused: boolean, color: string } 的函数返回一个 React.Node，以显示在选项卡栏中。未定义时，将使用场景 title。要隐藏，请参阅上一节中的 tabBarOptions.showLabel。

# tabBarBadge

在标签图标上的角标中显示的文本。接受 string 或 number。

# tabBarBadgeStyle

标签图标上的角标样式。您可以在此处指定背景颜色或文本颜色。

# tabBarButton

该函数返回一个 React 元素以渲染为选项卡按钮。它包装图标和标签并实现 onPress。默认情况下渲染 TouchableWithoutFeedback。 tabBarButton：props => <TouchableOpacity {... props} /> 会改用 TouchableOpacity。

# tabBarAccessibilityLabel

选项卡按钮的辅助功能标签。当用户点击选项卡时，将由屏幕阅读器读取。如果您没有该标签的标签，建议您进行设置。

# tabBarTestID

在测试中找到此选项卡按钮的 ID。

# unmountOnBlur

离开该屏幕时是否应卸载该屏幕。卸载屏幕会重置屏幕中的任何本地状态以及屏幕中嵌套导航器的状态。默认为 false。

通常，我们不建议您启用此 prop，因为用户不希望在切换标签时丢失其导航历史记录。如果启用此 prop，请考虑这是否会为用户提供更好的体验。

# Events

navigator 可以在某些操作上发出事件。 支持的事件有：

# tabPress

当用户在选项卡栏中按当前屏幕的选项卡按钮时，将触发此事件。 默认情况下，按选项卡可做几件事：

  • 如果选项卡未聚焦，则按 Tab 键将使该选项卡聚焦
  • 如果选项卡已经聚焦：
    • 如果选项卡的屏幕渲染滚动视图，则可以使用 useScrollToTop 将其滚动到顶部
    • 如果选项卡的屏幕渲染堆栈导航器，则会在堆栈上执行 popToTop 操作

为了防止出现默认行为，您可以调用 event.preventDefault：

React.useEffect(() => {
  const unsubscribe = navigation.addListener("tabPress", (e) => {
    // Prevent default behavior
    e.preventDefault();

    // Do something manually
    // ...
  });

  return unsubscribe;
}, [navigation]);

如果您有一个自定义 tab bar，请确保发出此事件。

# tabLongPress

当用户长时间按下选项卡栏中当前屏幕的选项卡按钮时，将触发此事件。 如果您有一个自定义 tab bar，请确保发出此事件。

例子：

React.useEffect(() => {
  const unsubscribe = navigation.addListener("tabLongPress", (e) => {
    // Do something
  });

  return unsubscribe;
}, [navigation]);

# Helpers

标签导航器将以下方法添加到导航 prop：

# jumpTo

导航到选项卡导航器中的现有屏幕。 该方法接受以下参数：

  • name - string - 跳转到的路由名称。
  • params - object - 合并到目标路由的屏幕参数（通过 route.params 在推送的屏幕中找到）。
navigation.jumpTo("Profile", { owner: "Michaś" });

# Example

import { createBottomTabNavigator } from "@react-navigation/bottom-tabs";
import MaterialCommunityIcons from "react-native-vector-icons/MaterialCommunityIcons";

const Tab = createBottomTabNavigator();

function MyTabs() {
  return (
    <Tab.Navigator
      initialRouteName="Feed"
      tabBarOptions={{
        activeTintColor: "#e91e63",
      }}
    >
      <Tab.Screen
        name="Feed"
        component={Feed}
        options={{
          tabBarLabel: "Home",
          tabBarIcon: ({ color, size }) => (
            <MaterialCommunityIcons name="home" color={color} size={size} />
          ),
        }}
      />
      <Tab.Screen
        name="Notifications"
        component={Notifications}
        options={{
          tabBarLabel: "Updates",
          tabBarIcon: ({ color, size }) => (
            <MaterialCommunityIcons name="bell" color={color} size={size} />
          ),
          tabBarBadge: 3,
        }}
      />
      <Tab.Screen
        name="Profile"
        component={Profile}
        options={{
          tabBarLabel: "Profile",
          tabBarIcon: ({ color, size }) => (
            <MaterialCommunityIcons name="account" color={color} size={size} />
          ),
        }}
      />
    </Tab.Navigator>
  );
}
