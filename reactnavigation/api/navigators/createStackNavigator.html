# createStackNavigator

为您的应用提供一种在每个新屏幕都放在堆栈顶部的屏幕之间跳转的方式。

默认情况下，堆栈导航器配置为具有熟悉的 iOS 和 Android 外观：在 iOS 上，新屏幕从右侧滑入，在 Android 上从底部淡入。 在 iOS 上，堆栈导航器也可以配置为模式样式，其中屏幕从底部滑入。

要使用此导航器，请确保您具有@react-navigation/native 及其依赖项（遵循本指南），然后安装 @react-navigation/stack：

npm install @react-navigation/stack

# API 定义

要使用此导航器，请从@react-navigation/stack 导入它：

import { createStackNavigator } from "@react-navigation/stack";

const Stack = createStackNavigator();

function MyStack() {
  return (
    <Stack.Navigator>
      <Stack.Screen name="Home" component={Home} />
      <Stack.Screen name="Notifications" component={Notifications} />
      <Stack.Screen name="Profile" component={Profile} />
      <Stack.Screen name="Settings" component={Settings} />
    </Stack.Navigator>
  );
}

# Props

Stack.Navigator 组件接受以下 props：

# initialRouteName

在导航器首次加载时要渲染的路由的名称。

# screenOptions

用于导航器中的屏幕的默认选项。

# keyboardHandlingEnabled

如果为 false，则导航到新屏幕时，屏幕上的键盘不会自动关闭。默认为 true。

# mode

定义渲染和过渡的样式：

  • card - 使用标准的 iOS 和 Android 屏幕过渡。这是默认值。

  • modal - 这做两件事：

    • 除非指定，否则将 headerMode 设置为 screen 显示
    • 使屏幕从 iOS 底部的底部滑入，这是一种常见的 iOS 模式。

# headerMode

指定标题的渲染方式：

float - 渲染一个停留在顶部的标题，并在更改屏幕时进行动画处理。这是 iOS 上的常见模式。 screen - 每个屏幕都有一个附加的标题，标题随屏幕一起淡入和淡出。这是 Android 上的常见模式。 none - 不显示标题。建议使用 headerShown 选项代替，以获得更多的粒度。

# detachInactiveScreens

布尔值，用于指示是否应将非活动屏幕从视图层次结构中分离出来以节省内存。确保从 react-native-screens 调用 enableScreens 使其起作用。默认为 true。

# Options

以下选项可用于配置导航器中的屏幕：

标题＃ 可用作 headerTitle 的后备的字符串。

# title

返回 React 元素以显示为标题的函数。 它接受包含以下属性的对象作为参数：

  • mode - 标题栏模式 - float 或 screen
  • layout - 屏幕尺寸
  • insets - 在标题栏中使用的安全区域插图
  • scene - 包含 2 个属性：
    • route - 标题栏的路由对象
    • descriptor - 包含 navigation prop 和屏幕 options 的描述符
  • previous - 如果没有上一屏幕，则前一屏幕的 scene 对象将是未定义的
  • 标题栏的 navigation prop
  • styleInterpolator - 返回标题栏中各种元素的插值样式的函数。

使用自定义标题时，请确保还将 headerMode 设置为 screen（更多信息请参见下文）。

例子：

header: ({ scene, previous, navigation }) => {
  const { options } = scene.descriptor;
  const title =
    options.headerTitle !== undefined
      ? options.headerTitle
      : options.title !== undefined
      ? options.title
      : scene.route.name;

  return (
    <MyHeader
      title={title}
      leftButton={
        previous ? <MyBackButton onPress={navigation.goBack} /> : undefined
      }
      style={options.headerStyle}
    />
  );
};

要为导航器中的所有屏幕设置自定义标题，可以在导航器的 screenOptions 属性中指定此选项。

使用自定义标题栏时，有两点需要牢记：

# 将 headerMode 设置为 screen

默认情况下，有一个浮动标题栏可为 iOS 上的多个屏幕渲染标题栏。 这些标题包括动画，可以平滑地相互切换。

将 headerMode 属性设置为 screen 可以使屏幕的标题成为屏幕的一部分，因此您无需实现动画来单独对其进行动画处理。

如果要自定义标题动画的方式并希望将 headerMode 保持为 float，则可以在 scene.progress.current 和 scene.progress.next prop 上进行插值。 例如，以下将淡入淡出标题：

const progress = Animated.add(scene.progress.current, scene.progress.next || 0);

const opacity = progress.interpolate({
  inputRange: [0, 1, 2],
  outputRange: [0, 1, 0],
});

return (
  <Animated.View style={{ opacity }}>{/* Header content */}</Animated.View>
);
# 在 headerStyle 中指定高度

如果标题栏的高度与默认标题栏的高度不同，则可能由于测量异步而注意到毛刺。 明确指定高度将避免此类故障。

例子：

headerStyle: {
  height: 80, // Specify the height of your custom header
};

请注意，由于您可以控制自定义标题的样式，因此默认情况下不会将此样式应用于标题。 如果您还想将此样式应用于标题，请使用 prop 中的 scene.descriptor.options.headerStyle。

# headerShown

是显示还是隐藏屏幕的标题。 除非导航器上的 headerMode 属性设置为 none，否则默认情况下将显示标题。 将此设置为 false 将隐藏标题。

# headerTitle

返回标题栏要使用的 React 元素的字符串或函数。 默认为场景 title。 指定函数后，它将接收一个包含 allowFontScaling，style 和 children 属性的对象。 children 属性包含标题字符串。

# headerTitleAlign

如何对齐标题。 可能的值：

  • left
  • center

默认设置为在 iOS 上居中，在 Android 上为左。

# headerTitleAllowFontScaling

标题栏标题字体是否应缩放以符合“文本大小”辅助功能设置。默认为 false。

# headerBackAllowFontScaling

后退按钮标题字体是否应缩放以符合“文本大小”辅助功能设置。默认为 false。

# headerBackAccessibilityLabel

标题栏后退按钮的辅助功能标签。

# headerBackImage

该函数返回一个 React 元素以在标题的后退按钮中显示自定义图像。使用函数时，它将在其参数对象中接收 tintColor。默认为带有背面图像源的 Image 组件，它是平台的默认背面图标图像（iOS 上为人字形，Android 上为箭头）。

# headerBackTitle

iOS 上的后退按钮使用的标题字符串。默认为上一个场景的 headerTitle。

# headerBackTitleVisible

为后退按钮标题是否可见提供了一个合理的默认值，但是如果您要覆盖它，则可以在此选项中使用 true 或 false。

# headerTruncatedBackTitle

当 headerBackTitle 不在屏幕上时，后退按钮使用的标题字符串。默认情况下为“Back”。

# headerRight

返回 React 元素以显示在标题右侧的函数。

# headerLeft

返回 React 元素以显示在标题左侧的函数。使用函数时，函数在渲染时会接收许多参数（onPress，label，labelStyle 等-请检查 types.tsx (opens new window) 以获取完整列表）。

默认情况下，使用 HeaderBackButton 组件。您可以实现它，并使用它来覆盖后退按钮的按下，例如：

import { HeaderBackButton } from "@react-navigation/stack";

// ...

<Screen
  name="Home"
  component={HomeScreen}
  options={{
    headerLeft: (props) => (
      <HeaderBackButton
        {...props}
        onPress={() => {
          // Do something
        }}
      />
    ),
  }}
/>;

# headerStyle

标题栏的样式对象。例如，您可以在此处指定自定义背景色。

# headerTitleStyle

标题组件的样式对象

# headerBackTitleStyle

返回标题的样式对象

# headerLeftContainerStyle

自定义 headerLeft 组件的容器的样式，例如添加填充。

# headerRightContainerStyle

自定义 headerRight 组件的容器的样式，例如添加填充。

# headerTitleContainerStyle

自定义 headerTitle 组件的容器的样式，例如添加填充。

默认情况下，headerTitleContainerStyle 具有绝对位置样式，并且向左和向右偏移。如果使用自定义的 headerLeft，这可能会导致空白或 headerLeft 和 headerTitle 之间的重叠。可以通过调整 headerTitleContainerStyle 中的左右样式和 headerTitleStyle 中的 marginHorizo​​ntal 来解决。

# headerTintColor

标题栏的文字颜色

# headerPressColorAndroid

material ripple 的颜色（仅适用于 Android> = 5.0）

# headerTransparent

默认为 false。如果为 true，则标题将没有背景，除非您显式为其提供 headerBackground。标题栏也将浮动在屏幕上，使其与下面的内容重叠。

如果要渲染半透明的标题或模糊的背景，这很有用。

请注意，如果您不希望内容出现在标题下，则需要手动为内容添加一个上边距。 React Navigation 不会自动执行。

要获得标题的高度，可以将 HeaderHeightContext 与 React 的 Context API 一起使用或 useHeaderHeight：

import { HeaderHeightContext } from '@react-navigation/stack';

// ...

<HeaderHeightContext.Consumer>
  {headerHeight => (
    /* render something */
  )}
</HeaderHeightContext.Consumer>

或

import { useHeaderHeight } from "@react-navigation/stack";

// ...

const headerHeight = useHeaderHeight();

# headerBackground

返回 React 元素以渲染为标题背景的函数。 这对于使用背景（例如图像或渐变）很有用。

例如，可以将其与 headerTransparent 一起使用以渲染模糊视图以创建半透明的标题。

import { BlurView } from "expo-blur";

// ...

<Screen
  name="Home"
  component={HomeScreen}
  options={{
    headerTransparent: true,
    headerBackground: () => (
      <BlurView tint="light" intensity={100} style={StyleSheet.absoluteFill} />
    ),
  }}
/>;

# headerStatusBarHeight

在标题顶部添加额外的填充以说明半透明的状态栏。默认情况下，它使用设备安全区域插图中的最大值。传递 0 或自定义值以禁用默认行为，并自定义高度。

# cardShadowEnabled

使用此 prop 在过渡期间具有可见的阴影。默认为 true。

# cardOverlayEnabled

使用此 prop 可以在过渡期间在卡下看到半透明的深色覆盖层。在 Android 上默认为 true，在 iOS 上默认为 false。

# cardOverlay

该函数返回一个 React Element 来显示为 card 的叠加层。使用此功能时，请确保将 cardOverlayEnabled 设置为 true。

# cardStyle

堆栈中 card 的样式对象。您可以在此处提供要使用的自定义背景颜色，而不是默认背景颜色。

您也可以指定 {backgroundColor：'transparent'}，使前一个屏幕在下面可见（对于透明模式）。这对于实现模态对话框之类的东西很有用。当使用透明背景时，还应该在堆栈视图配置中指定mode: 'modal'，这样以前的屏幕就不会分离并且在下面保持可见状态。

# animationEnabled

屏幕上是否应启用过渡动画。如果将其设置为 false，则按下或弹出时屏幕不会动画。在 iOS 和 Android 上默认为 true，在 Web 上默认为 false。

# animationTypeForReplace

当此屏幕替换另一个屏幕时要使用的动画类型。它采用以下值：

  • push - 将使用被 push 的新屏幕的动画
  • pop - 将使用正在 pop 的屏幕的动画

默认为 push。

使用 pop 时，pop 动画将应用于要替换的屏幕。

# poseEnabled

是否可以使用手势关闭此屏幕。在 iOS 上默认为 true，在 Android 上默认为 false。

Web 不支持手势。

# gestureResponseDistance

从屏幕边缘开始覆盖触摸距离的对象以识别手势。该对象可以包含以下属性：

  • horizontal - number -水平方向的距离。默认为 50。
  • vertical - number -垂直方向的距离。默认值为 135。

Web 不支持此功能。

# gestureVelocityImpact

确定手势速度相关性的数字。默认值为 0.3。

Web 不支持此功能。

# gestureDirection

手势的方向。有关详细信息，请参见 “动画” (opens new window) 部分。

Web 不支持此功能。

# transitionSpec

屏幕过渡的配置对象。有关详细信息，请参见 “动画” (opens new window) 部分。

# cardStyleInterpolator

插卡各个部分的插补样式。有关详细信息，请参见 “动画” (opens new window) 部分。

# headerStyleInterpolator

标题栏各个部分的内插样式。有关详细信息，请参见 “动画” (opens new window) 部分。

# detachPreviousScreen

布尔值，用于指示是否从视图层次结构中分离上一个屏幕以节省内存。如果您需要通过活动屏幕查看前一个屏幕，请将其设置为 false。仅当 detachInactiveScreens 未设置为 false 时适用。当 mode ='modal' 时，对于最后一个屏幕默认为 false，否则为 true。

# safeAreaInsets

屏幕的安全区域插图。 这是为了避免使用诸如缺口和状态栏之类的元素。 默认情况下，将自动检测设备的安全区域插图。 您可以使用此选项覆盖行为。

接受包含以下可选属性的对象：

  • top - number - 顶部插图的值，例如 包含状态栏和刻痕的区域。
  • right - number - 左插图的值。
  • bottom - number - 顶部插图的值，例如 底部的区域导航栏。
  • left - number - 右插图的值。

# Events

导航器可以在某些操作上发出事件。 支持的事件有：

# transitionStart

当前屏幕的过渡动画开始时将触发此事件。

例子：

React.useEffect(() => {
  const unsubscribe = navigation.addListener("transitionStart", (e) => {
    // Do something
  });

  return unsubscribe;
}, [navigation]);

# transitionEnd

当当前屏幕的过渡动画结束时，将触发此事件。

例子：

React.useEffect(() => {
  const unsubscribe = navigation.addListener("transitionEnd", (e) => {
    // Do something
  });

  return unsubscribe;
}, [navigation]);

# Helpers

堆栈导航器将以下方法添加到导航 prop：

# push

将新屏幕 push 堆栈顶部并导航到它。 该方法接受以下参数：

  • name - string - 推入堆栈的路由的名称。
  • params - object - 合并到目标路由的屏幕参数（通过 route.params 在推送的屏幕中找到）。
navigation.push("Profile", { owner: "Michaś" });

# pop

从堆栈中 pop 当前屏幕，然后导航回到上一个屏幕。 它带有一个可选参数（计数），它允许您指定弹出多少个屏幕。

navigation.pop();

# popToTop

pop 堆栈中除第一个屏幕外的所有屏幕，并导航至该屏幕。

navigation.popToTop();

栗子

import { createStackNavigator } from "@react-navigation/stack";

const Stack = createStackNavigator();

function MyStack() {
  return (
    <Stack.Navigator
      initialRouteName="Home"
      headerMode="screen"
      screenOptions={{
        headerTintColor: "white",
        headerStyle: { backgroundColor: "tomato" },
      }}
    >
      <Stack.Screen
        name="Home"
        component={Home}
        options={{
          title: "Awesome app",
        }}
      />
      <Stack.Screen
        name="Profile"
        component={Profile}
        options={{
          title: "My profile",
        }}
      />
      <Stack.Screen
        name="Settings"
        component={Settings}
        options={{
          gestureEnabled: false,
        }}
      />
    </Stack.Navigator>
  );
}

# 动画(Animations)

# 动画相关选项

当添加或移除屏幕时，Stack Navigator 会显示各种选项来配置过渡动画。通过在每个屏幕的选项属性中指定选项，可以在每个屏幕上自定义这些过渡动画。

  • gestureDirection - 滑动手势的方向：
    • horizontal - 关闭屏幕的手势将从 RTL 中的左侧和右侧开始。对于动画，屏幕将使用 SlideFromRightIOS 从右侧滑动，而在 RTL 中从左侧滑动。
    • horizontal-inverted - 在 RTL 中，关闭屏幕的手势从右开始，从左开始。对于动画，屏幕将通过 SlideFromRightIOS 从左侧滑动，而在 RTL 中，当方向反转时将从右侧滑动。
    • vertical - 关闭屏幕的手势将从顶部开始。对于动画，屏幕将从底部滑动。
    • vertical-inverted - 关闭屏幕的手势将从底部开始。对于动画，屏幕将从顶部滑动。

您可能还需要指定一个匹配的水平/垂直动画以及 gestureDirection。对于库中包含的动画，如果将手放方向设置为倒转的其中之一，它还将翻转动画方向。

  • transitionSpec - 一个对象，它指定动画类型（定时或弹跳）及其选项（例如计时的持续时间）。它具有 2 个属性：

    • open - 添加屏幕时过渡的配置

    • close - 删除屏幕时过渡的配置。

      每个对象应指定 2 个属性：

    • animation - 用于动画的动画功能。支持的值是 timing 和 spring 。

    • config - timing 功能的配置对象。对于 timing，可以是 duration 和 easing。对于 spring，可以是 stiffness，damping，mass，overshootClamping，restDisplacementThreshold 和 restSpeedThreshold。

使用 spring 动画的配置如下所示：

const config = {
  animation: "spring",
  config: {
    stiffness: 1000,
    damping: 500,
    mass: 3,
    overshootClamping: true,
    restDisplacementThreshold: 0.01,
    restSpeedThreshold: 0.01,
  },
};

我们可以在 transitionSpec 选项中传递此配置：

<Stack.Screen
  name="Profile"
  component={Profile}
  options={{
    transitionSpec: {
      open: config,
      close: config,
    },
  }}
/>

  • cardStyleInterpolator - 这是一个用于为 card 的各个部分指定插值样式的函数。这使您可以在从一个屏幕导航到另一个屏幕时自定义过渡。期望至少返回空对象，其中可能包含容器的插值样式，卡片本身，覆盖和阴影。支持的属性是：

    • containerStyle - 包装 card 的容器视图的样式。

    • cardStyle - 表示 card 的视图的样式。

    • overlayStyle - 表示下面的半透明叠加层的视图的样式

    • shadowStyle - 表示 card 阴影的视图样式。

      该函数在其参数中接收以下属性：

    • current - 当前屏幕的值：

      • progress - 表示当前屏幕进度值的动画节点。

    • next - 堆栈中此值之后的屏幕值。如果屏幕动画是最后一个动画，则可以将其定义为未定义。

      • progress - 动画节点，表示下一个屏幕的进度值。

    • index - 堆栈中 card 的索引。

    • closing - 动画节点，表示 card 是否正在关闭。关闭时为 1，否则为 0。

    • layouts - 我们用于动画的各种项目的布局尺寸。

      • screen - 整个屏幕的布局。包含高度和宽度属性。

请注意，当一个屏幕不是最后一个屏幕时，它将使用下一个屏幕的过渡配置。 这是因为许多过渡都涉及上一个屏幕的动画，因此这两个过渡必须保持在一起，以防止在两个屏幕上运行两种不同类型的过渡（例如，幻灯片和模态）。 您可以检查下一个参数，以了解是否要对上一个屏幕进行动画处理。 有关此参数的更多信息，请参见动画部分。

只会使屏幕淡入淡出 (fades) 的配置如下所示：

const forFade = ({ current }) => ({
  cardStyle: {
    opacity: current.progress,
  },
});

我们可以在 cardStyleInterpolator 选项中传递此函数：

<Stack.Screen
  name="Profile"
  component={Profile}
  options={{ cardStyleInterpolator: forFade }}
/>

将为每个屏幕调用 interpolator。 例如，假设您在堆栈中有 2 个屏幕，A 和 B。B 是新的屏幕，而 A 是前一个屏幕。 将为每个屏幕调用 interpolator：

  • interpolator 被称为 B：在这里，current.progress 值表示过渡的进度，该过渡将从 0 开始到 1 结束。因为 B 是最后一个屏幕，所以将没有 next.progress。
  • interpolator 被要求为 A：在这里，current.progress 将保持为 1 的值，并且不会改变，因为当前过渡是为 B（而不是 A）运行的。next.progress 值表示 B 的进度，并且将为 从 0 开始到 1 结束。

假设我们要在过渡期间为两个屏幕设置动画。 最简单的方法是合并当前屏幕和下一屏幕的进度值：

const progress = Animated.add(
  current.progress.interpolate({
    inputRange: [0, 1],
    outputRange: [0, 1],
    extrapolate: "clamp",
  }),
  next
    ? next.progress.interpolate({
        inputRange: [0, 1],
        outputRange: [0, 1],
        extrapolate: "clamp",
      })
    : 0
);

在这里，屏幕 A 将同时具有 current.progress 和 next.progress，并且由于 current.progress 保持为 1，并且 next.progress 正在更改，因此，进度将从 1 更改为 2。屏幕 B 将仅具有 current.progress 会从 0 变为 1。因此，我们可以对 0-1 和 1-2 应用不同的插值，以分别为聚焦屏幕和非聚焦屏幕设置动画。

将前一个屏幕稍微向左平移，并从右边缘平移当前屏幕的配置如下所示：

const forSlide = ({ current, next, inverted, layouts: { screen } }) => {
  const progress = Animated.add(
    current.progress.interpolate({
      inputRange: [0, 1],
      outputRange: [0, 1],
      extrapolate: "clamp",
    }),
    next
      ? next.progress.interpolate({
          inputRange: [0, 1],
          outputRange: [0, 1],
          extrapolate: "clamp",
        })
      : 0
  );

  return {
    cardStyle: {
      transform: [
        {
          translateX: Animated.multiply(
            progress.interpolate({
              inputRange: [0, 1, 2],
              outputRange: [
                screen.width, // Focused, but offscreen in the beginning
                0, // Fully focused
                screen.width * -0.3, // Fully unfocused
              ],
              extrapolate: "clamp",
            }),
            inverted
          ),
        },
      ],
    },
  };
};

  • headerStyleInterpolator - 这是一个函数，用于为标题的各个部分指定插值样式。期望至少返回空对象，其中可能包含左标签和按钮，右按钮，标题和背景的插值样式。支持的属性是：

    • leftLabelStyle - 左按钮标签（后退按钮标签）的样式。

    • leftButtonStyle - 左按钮（通常是后退按钮）的样式。

    • rightButtonStyle - 右键的样式。

    • titleStyle - 标题栏标题文本的样式。

    • backgroundStyle - 标题栏背景的样式。 该函数在其参数中接收以下属性：

    • current - 当前屏幕（拥有此标题的屏幕）的值。

      • progress - 表示当前屏幕进度值的动画节点。屏幕应开始显示时为 0，中途显示为 0.5，完全显示时为 1。

    • next - 堆栈中此值之后的屏幕值。如果屏幕动画是最后一个动画，则可以将其定义为未定义。

      • progress - 动画节点，表示下一个屏幕的进度值。

    • layouts - 我们用于动画的各种项目的布局尺寸。每个布局对象都包含 height 和 width 属性。

      • screen - 整个屏幕的布局。
      • title - 标题元素的布局。不渲染标题时可能未定义。
      • leftLabel - 后退按钮标签的布局。当不渲染后退按钮标签时，可能未定义。

只是淡入淡出元素的配置如下所示：

const forFade = ({ current, next }) => {
  const opacity = Animated.add(
    current.progress,
    next ? next.progress : 0
  ).interpolate({
    inputRange: [0, 1, 2],
    outputRange: [0, 1, 0],
  });

  return {
    leftButtonStyle: { opacity },
    rightButtonStyle: { opacity },
    titleStyle: { opacity },
    backgroundStyle: { opacity },
  };
};

我们可以在 headerStyleInterpolator 选项中传递此函数：

<Stack.Screen
  name="Profile"
  component={Profile}
  options={{ headerStyleInterpolator: forFade }}
/>

# 预制配置

使用这些选项，可以为屏幕构建自定义过渡动画。 我们还使用现成的动画从库中导出各种配置，您可以使用这些动画：

# TransitionSpecs
  • TransitionIOSSpec - UINavigationController 的动画配置中的确切值。
  • FadeInFromBottomAndroidSpec - 来自 Android Nougat 的活动打开动画的配置。
  • FadeOutToBottomAndroidSpec - 来自 Android Nougat 的活动关闭动画的配置。
  • RevealFromBottomAndroidSpec - 来自 Android Pie 的活动打开动画的近似配置。

例子：

import { TransitionSpecs } from "@react-navigation/stack";

// ...

<Stack.Screen
  name="Profile"
  component={Profile}
  options={{
    transitionSpec: {
      open: TransitionSpecs.TransitionIOSSpec,
      close: TransitionSpecs.TransitionIOSSpec,
    },
  }}
/>;
# CardStyleInterpolators
  • forHorizontalIOS - 右侧的标准 iOS 样式幻灯片。
  • forVerticalIOS - 从底部插入标准 iOS 样式的幻灯片（用于模式）。
  • forModalPresentationIOS - iOS 13 中的标准 iOS 样式模态动画。
  • forFadeFromBottomAndroid - Android Oreo 从底部开始采用标准的 Android 样式淡入。
  • forRevealFromBottomAndroid - 标准的 Android 样式，从底部开始显示 Android Pie。

Android Oreo 样式垂直屏幕淡入淡出动画的示例配置：

import { CardStyleInterpolators } from "@react-navigation/stack";

// ...

<Stack.Screen
  name="Profile"
  component={Profile}
  options={{
    title: "Profile",
    cardStyleInterpolator: CardStyleInterpolators.forFadeFromBottomAndroid,
  }}
/>;
# HeaderStyleInterpolators
  • forUIKit - 标题栏的标准 UIKit 样式动画，标题淡入后退按钮标签。
  • forFade - 标题元素的简单淡入淡出动画。
  • forStatic - 简单的转换动画，可将标题与滑动屏幕一起转换。

标题元素进入后退按钮的标题元素的默认 iOS 动画的示例配置：

import { HeaderStyleInterpolators } from "@react-navigation/stack";

// ...

<Stack.Screen
  name="Profile"
  component={Profile}
  options={{
    title: "Profile",
    headerStyleInterpolator: HeaderStyleInterpolators.forUIKit,
  }}
/>;

注意：始终在文件的顶层定义动画配置，以确保引用在重新渲染时不会发生变化。这对于平稳可靠的过渡动画非常重要。

# TransitionPresets

我们导出各种过渡预设，这些过渡预设将这些选项的各种集合捆绑在一起以匹配某些本地动画。过渡预设是一个对象，其中包含一些在 TransitionPresets 下导出的与动画相关的屏幕选项。当前，以下预设可用：

  • SlideFromRightIOS - 标准的 iOS 导航过渡。
  • ModalSlideFromBottomIOS - 模式的标准 iOS 导航过渡。
  • ModalPresentationIOS - 标准的 iOS 模态表示样式（在 iOS 13 中引入）。
  • FadeFromBottomAndroid - 在 Android <9（Oreo）上打开或关闭活动时，标准的 Android 导航过渡。
  • RevealFromBottomAndroid - 在 Android 9（Pie）上打开或关闭活动时的标准 Android 导航过渡。
  • ScaleFromCenterAndroid - 在大于等于 10 的 Android 上打开或关闭活动时的标准 Android 导航过渡。
  • DefaultTransition - 当前平台的默认导航过渡。
  • ModalTransition - 当前平台的默认模式过渡。

您可以在 options 中散布这些预设，以自定义屏幕的动画：

import { TransitionPresets } from "@react-navigation/stack";

// ...

<Stack.Screen
  name="Profile"
  component={Profile}
  options={{
    title: "Profile",
    ...TransitionPresets.ModalSlideFromBottomIOS,
  }}
/>;

如果要为导航器中的所有屏幕自定义过渡动画，则可以在导航器的 screenOptions 属性中指定它。

iOS 模态表示样式的示例配置：

import { TransitionPresets } from "@react-navigation/stack";

// ...

<Stack.Navigator
  initialRouteName="Home"
  screenOptions={({ route, navigation }) => ({
    headerShown: false,
    gestureEnabled: true,
    cardOverlayEnabled: true,
    headerStatusBarHeight:
      navigation
        .dangerouslyGetState()
        .routes.findIndex((r) => r.key === route.key) > 0
        ? 0
        : undefined,
    ...TransitionPresets.ModalPresentationIOS,
  })}
  mode="modal"
>
  <Stack.Screen name="Home" component={Home} />
  <Stack.Screen name="Profile" component={Profile} />
</Stack.Navigator>;

注意：需要为整个堆栈配置 ModalPresentationIOS 预设，以使其正常工作。 如果您希望几个屏幕具有此过渡，则可以在此过渡的根处添加一个模式堆栈，并将常规堆栈嵌套在其中。

# 透明模态

透明模态类似于覆盖屏幕的模态对话框。 前一个屏幕在下面仍然保持可见。 要获得透明的模式屏幕，通常更容易创建单独的模式堆栈。 在模式堆栈中，您将需要配置一些东西：

  • 将模式 prop 设置为 modal，将最后一个屏幕的 detachPreviousScreen 选项设置为 false
  • 使用 cardStyle 将卡片背景设置为透明
  • 使用自定义动画代替默认的平台动画（在这种情况下，我们将使用淡入淡出）
  • 使用 headerShown 禁用标题：false（可选）
  • 使用 cardOverlayEnabled 启用覆盖：true（可选）

例子：

<Stack.Navigator
  screenOptions={{
    headerShown: false,
    cardStyle: { backgroundColor: "transparent" },
    cardOverlayEnabled: true,
    cardStyleInterpolator: ({ current: { progress } }) => ({
      cardStyle: {
        opacity: progress.interpolate({
          inputRange: [0, 0.5, 0.9, 1],
          outputRange: [0, 0.25, 0.7, 1],
        }),
      },
      overlayStyle: {
        opacity: progress.interpolate({
          inputRange: [0, 1],
          outputRange: [0, 0.5],
          extrapolate: "clamp",
        }),
      },
    }),
  }}
  mode="modal"
>
  <Stack.Screen name="Home" component={HomeStack} />
  <Stack.Screen name="Modal" component={ModalScreen} />
</Stack.Navigator>

现在，当您导航至 “Modal” 屏幕时，其下方仍将显示 “Home” 屏幕。
