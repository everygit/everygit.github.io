# StackActions

StackActions 是一个对象，其中包含用于生成特定于基于堆栈的导航器的操作的方法。 它的方法扩展了 CommonActions 中可用的操作。

支持以下操作：

# replace

replace 动作允许在导航状态下替换路由。 它采用以下参数：

  • name - string - 已在某处注册的路由的目的地名称。
  • params - object - 合并到目标路由的参数。
import { StackActions } from "@react-navigation/native";

navigation.dispatch(
  StackActions.replace("Profile", {
    user: "jane",
  })
);

如果要替换特定路由，则可以添加引用路由键的 source 属性和引用导航状态键的 target 属性：

import { StackActions } from "@react-navigation/native";

navigation.dispatch({
  ...StackActions.replace("Profile", {
    user: "jane",
  }),
  source: route.key,
  target: navigation.dangerouslyGetState().key,
});

如果将 source 属性显式设置为 undefined，它将替换聚焦的路由。

# push

push 动作将一条路由添加到堆栈的顶部并向前 navigate。 这与 navigate 不同，如果给定名称的路由已经存在，导航将跳回到堆栈中的较早位置。 push 将始终添加在最前面，因此一条路线可以多次显示。

  • name - string - 推入堆栈的路由的名称。
  • params - object - 合并到目标路由的屏幕参数（通过 route.params 在推送的屏幕中找到）。

如果要替换特定路由，则可以添加引用路由键的 source 属性和引用导航状态键的 target 属性：

import { StackActions } from "@react-navigation/native";

const pushAction = StackActions.push("Profile", { user: "Wojtek" });

navigation.dispatch(pushAction);

# pop

pop 操作将您带回到堆栈中的上一个屏幕。 它带有一个可选参数（ count ），它允许您指定弹出多少个屏幕。

import { StackActions } from "@react-navigation/native";

const popAction = StackActions.pop(1);

navigation.dispatch(popAction);

# popToTop

popToTop 操作将您带回到堆栈中的第一个屏幕，关闭所有其他屏幕。 它在功能上与 StackActions.pop({n: currentIndex}) 相同。

import { StackActions } from "@react-navigation/native";

navigation.dispatch(StackActions.popToTop());
