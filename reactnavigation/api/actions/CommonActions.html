# CommonActions

导航动作是至少包含 type 属性的对象。在内部，该动作可以由路由器使用 getStateForAction 方法处理，以从现有导航状态返回新状态。

每个导航操作可以至少包含以下属性：

  • type（required）- 表示操作名称的字符串。
  • 有效负载（options）- 包含有关操作的其他信息的对象。例如，它将包含用于导航的名称和参数。
  • 源（optional）- 路径的关键字，应将其视为操作的来源。这用于某些操作，以确定要在其上应用该操作的路由。默认情况下，navigation.dispatch 添加调度该动作的路由的键。
  • target（optional）- 应该对其执行操作的导航状态的键。

重要的是要强调指出，未处理动作时调度导航动作不会引发任何错误（类似于您在 redux 中调度未由 reducer 处理的动作且什么也没有发生的情况）。

# Common actions

该库在 CommonActions 命名空间下导出了几个动作创建者。您应该使用这些动作创建者，而不是手动编写动作对象。

# navigate

navigate 操作允许导航到特定路由。它采用以下参数：

  • name - string - 已在某处注册的路由的目的地名称。
  • key - string - 导航到的路由的标识符。导航回到该路由（如果已存在）。
  • params - object - 合并到目标路由的参数。

传递的 options 对象至少应包含一个 key 或 name 属性，以及可选的 params。如果同时传递了 key 和 name，则如果找不到匹配项，则堆栈导航器将使用指定的键创建新的路由。

import { CommonActions } from "@react-navigation/native";

navigation.dispatch(
  CommonActions.navigate({
    name: "Profile",
    params: {
      user: "jane",
    },
  })
);

在堆栈导航器中，使用屏幕名称调用 navigate 会导致不同的行为，具体取决于屏幕是否存在。 如果该屏幕已存在于堆栈的历史记录中，它将返回该屏幕并在此之后删除所有屏幕。 如果屏幕不存在，它将推送一个新屏幕。

默认情况下，屏幕由其名称标识。 但是，您也可以使用 getId 道具对其进行自定义，以将这些参数考虑在内。

# reset

重置操作允许将导航状态重置为给定状态。 它采用以下参数：

  • state - object - 要使用的新导航状态对象。
import { CommonActions } from "@react-navigation/native";

navigation.dispatch(
  CommonActions.reset({
    index: 1,
    routes: [
      { name: "Home" },
      {
        name: "Profile",
        params: { user: "jane" },
      },
    ],
  })
);

在 reset 中指定的状态对象将现有的导航状态替换为新的状态。 这意味着，如果您提供没有键的新路由对象，或使用其他键的路由对象，则会删除这些路由的现有屏幕并添加新屏幕。

如果要保留现有屏幕，而只想修改状态，则可以传递一个函数来 dispatch 可以获取现有状态的位置。 然后，您可以根据需要进行更改（确保不更改现有状态，而是为更改创建新的状态对象）。 并返回具有所需状态的 reset 操作：

import { CommonActions } from "@react-navigation/native";

navigation.dispatch((state) => {
  // Remove the home route from the stack
  const routes = state.routes.filter((r) => r.name !== "Home");

  return CommonActions.reset({
    ...state,
    routes,
    index: routes.length - 1,
  });
});

注意：认为导航器的状态对象是内部的，并且在次要版本中可能会发生更改。 除非确实需要，否则请避免使用导航状态状态对象中的属性（索引和路由除外）。 如果有某些功能不能不依靠状态对象的结构来实现，请提出一个问题。

# goBack

使用 goBack 动作创建器可以返回历史记录中的上一条路由。 它不需要任何参数。

import { CommonActions } from "@react-navigation/native";

navigation.dispatch(CommonActions.goBack());

如果要从特定路径返回，可以添加引用路径键的 source 属性和引用包含该路径的导航器 key 的 target 属性：

import { CommonActions } from "@react-navigation/native";

navigation.dispatch({
  ...CommonActions.goBack(),
  source: route.key,
  target: state.key,
});

默认情况下，调度动作的路由的键作为 source 属性传递，而 target 属性 undefined。

# setParams

setParams 操作允许更新特定路由的参数。 它采用以下参数：

  • params - object - required - 要合并到现有路径参数中的新参数。
import { CommonActions } from "@react-navigation/native";

navigation.dispatch(CommonActions.setParams({ user: "Wojtek" }));

如果要为特定路由设置参数，则可以添加引用路由键的 source 属性：

import { CommonActions } from "@react-navigation/native";

navigation.dispatch({
  ...CommonActions.setParams({ user: "Wojtek" }),
  source: route.key,
});

如果将 source 属性显式设置为 undefined，它将为重点路由设置参数。
