# 服务端渲染

注意:此 API 是实验性的，可能会在小版本中更改。

本指南将介绍如何使用 React Native Web 和 React Navigation 服务器渲染你的 React Native 应用。我们将讨论以下情况:

  1. 根据请求 URL 渲染正确的布局
  2. 根据聚焦屏幕设置适当的页面元数据

# 先决条件

在你遵循指南之前，请确保你的应用程序已经在服务器上渲染良好。要做到这一点，你需要确保以下几点:

  • 你使用的所有依赖项在发布到 npm 之前都会被编译，所以 Node 上不会出现语法错误。
  • Node 被配置为能够 require 图像和字体等资产文件。您可以尝试 webpack-isomorphic-tools (opens new window) 来实现这一点。
  • react-native 的别名是 react-native-web。您可以通过 babel-plugin-module-resolver (opens new window) 来实现。

注意:React Navigation 中的一些库在 Web 上不能很好地工作，比如 @react-navigation/material-top-tabs。如果你使用的是 Expo 库，SSR 也不起作用。

# 渲染 app

首先，让我们看一个例子，你如何做服务器渲染与 React Native Web 不涉及 React 导航:

import { AppRegistry } from "react-native-web";
import ReactDOMServer from "react-dom/server";
import App from "./src/App";

const { element, getStyleElement } = AppRegistry.getApplication("App");

const html = ReactDOMServer.renderToString(element);
const css = ReactDOMServer.renderToStaticMarkup(getStyleElement());

const document = `
  <!DOCTYPE html>
  <html style="height: 100%">
  <meta charset="utf-8">
  <meta httpEquiv="X-UA-Compatible" content="IE=edge">
  <meta
    name="viewport"
    content="width=device-width, initial-scale=1, minimum-scale=1, maximum-scale=1.00001, viewport-fit=cover"
  >
  ${css}
  <body style="min-height: 100%">
  <div id="root" style="display: flex; min-height: 100vh">
  ${html}
  </div>
`;

在这里，./src/App 是你有 apgistry 的文件。AppRegistry.registerComponent('App', () => App)。

如果你在你的应用程序中使用 React Navigation，这将渲染你的主页所渲染的屏幕。但是，如果你在你的应用中配置了链接，你会想要在服务器上为请求 URL 呈现正确的屏幕，以便它匹配将在客户端上呈现的内容。

我们可以通过在 location 中传递这个信息来使用 ServerContainer 来做到这一点。例如，在 Koa 中，你可以使用 context 参数中的 path 和 search 属性:

app.use(async (ctx) => {
  const location = new URL(ctx.url, "https://example.org/");

  const { element, getStyleElement } = AppRegistry.getApplication("App");

  const html = ReactDOMServer.renderToString(
    <ServerContainer location={location}>{element}</ServerContainer>
  );

  const css = ReactDOMServer.renderToStaticMarkup(getStyleElement());

  const document = `
    <!DOCTYPE html>
    <html style="height: 100%">
    <meta charset="utf-8">
    <meta httpEquiv="X-UA-Compatible" content="IE=edge">
    <meta
      name="viewport"
      content="width=device-width, initial-scale=1, minimum-scale=1, maximum-scale=1.00001, viewport-fit=cover"
    >
    ${css}
    <body style="min-height: 100%">
    <div id="root" style="display: flex; min-height: 100vh">
    ${html}
    </div>
`;

  ctx.body = document;
});

你可能还想为搜索引擎、打开图表等设置正确的文档标题和描述。为此，您可以向容器传递一个ref，它将为您提供当前屏幕的选项。

app.use(async (ctx) => {
  const location = new URL(ctx.url, 'https://example.org/');

  const { element, getStyleElement } = AppRegistry.getApplication('App');

  const ref = React.createRef<ServerContainerRef>();

  const html = ReactDOMServer.renderToString(
    <ServerContainer
      ref={ref}
      location={location}
    >
      {element}
    </ServerContainer>
  );

  const css = ReactDOMServer.renderToStaticMarkup(getStyleElement());

  const options = ref.current?.getCurrentOptions();

  const document = `
    <!DOCTYPE html>
    <html style="height: 100%">
    <meta charset="utf-8">
    <meta httpEquiv="X-UA-Compatible" content="IE=edge">
    <meta
      name="viewport"
      content="width=device-width, initial-scale=1, minimum-scale=1, maximum-scale=1.00001, viewport-fit=cover"
    >
    ${css}
    <title>${options.title}</title>
    <body style="min-height: 100%">
    <div id="root" style="display: flex; min-height: 100vh">
    ${html}
    </div>
`;

  ctx.body = document;
});

确保你已经在屏幕中指定了一个 title 选项:

<Stack.Screen
  name="Profile"
  component={ProfileScreen}
  options={{ title: "My profile" }}
/>

# 处理 404 或其他状态代码

当渲染无效 URL 的屏幕时，我们还应该从服务器返回一个 404 状态代码。 首先，我们需要创建一个附加状态代码的上下文。为了做到这一点，将以下代码放在一个单独的文件中，我们将在服务器和客户端导入:

import * as React from "react";

const StatusCodeContext = React.createContext();

export default StatusCodeContext;

然后，我们需要在 NotFound 屏幕中使用上下文。在这里，我们添加了一个值为 404 的 code 来表示没有找到屏幕:

function NotFound() {
  const status = React.useContext(StatusCodeContext);

  if (status) {
    staus.code = 404;
  }

  return (
    <View>
      <Text>Oops! This URL doesn't exist.</Text>
    </View>
  );
}

如果需要，还可以在该对象中附加其他信息。

接下来，我们需要创建一个状态对象，以便在服务器上的上下文中传递。默认情况下，我们将把code设置为 200。然后在 StatusCodeContext.Provider 中传递该用 ServerContainer 包装元素对象:

// Create a status object
const status = { code: 200 };

const html = ReactDOMServer.renderToString(
  // Pass the status object via context
  <StatusCodeContext.Provider value={status}>
    <ServerContainer ref={ref} location={location}>
      {element}
    </ServerContainer>
  </StatusCodeContext.Provider>
);

// After rendering, get the status code and use it for server's response
ctx.status = status.code;

在我们用 ReactDOMServer.renderToString 渲染应用程序之后。如果 NotFound 屏幕被渲染，status 对象的 code 属性将被更新为 404。

对于其他状态码也可以采用类似的方法，例如 401 用于未授权的状态码等。

# 总结

  • 使用 ServerContainer 上的 location 来根据传入的请求渲染正确的屏幕。
  • 为当前屏幕的 ServerContainer 附加一个 ref来获取选项。
  • 使用上下文附加更多信息，如状态代码。
