# 使用 TypeScript 进行类型检查

React Navigation 是用 TypeScript 编写的，它为 TypeScript 项目导出类型定义。

# 导航器的类型检查

要检查路由名称和参数的类型，我们需要做的第一件事是创建一个对象类型，将路由名称映射到路由的参数。例如，假设我们在根导航中有一个名为 Profile 的路由，它应该有一个参数 userId:

type RootStackParamList = {
  Profile: { userId: string },
};

类似地，我们需要对每个路由做同样的事情:

type RootStackParamList = {
  Home: undefined,
  Profile: { userId: string },
  Feed: { sort: "latest" | "top" } | undefined,
};

指定 undefined 意味着路由没有参数。带有 undefined 的联合类型 (例如，SomeType | undefined) 意味着参数是可选的。

在定义了映射之后，我们需要告诉导航器使用它。为此，我们可以将它作为泛型传递给 createXNavigator 函数:

import { createStackNavigator } from '@react-navigation/stack';

const RootStack = createStackNavigator<RootStackParamList>();

然后我们可以使用它:

<RootStack.Navigator initialRouteName="Home">
  <RootStack.Screen name="Home" component={Home} />
  <RootStack.Screen
    name="Profile"
    component={Profile}
    initialParams={{ userId: user.id }}
  />
  <RootStack.Screen name="Feed" component={Feed} />
</RootStack.Navigator>

这将为 Navigator 和 Screen 组件的 props 提供类型检查和智能提示。

# 屏幕的类型检查

要检查屏幕，我们需要注解屏幕接收到的 navigation和 route。 要注解导航道具，我们需要从导航器中导入相应的类型。例如，@react-navigation/stack 中的 StackNavigationProp:

import { StackNavigationProp } from "@react-navigation/stack";

type ProfileScreenNavigationProp = StackNavigationProp<
  RootStackParamList,
  "Profile"
>;

type Props = {
  navigation: ProfileScreenNavigationProp,
};

navigation 的类型接受 2 个泛型，我们在前面定义的参数列表对象，以及当前路由的名称。这允许我们输入检查路由名称和参数，你正在使用 navigate，push 等。当你调用 setParams 时，当前路由的名称是必要的。

类似地，你可以从@react-navigation/drawer 中导入 DrawerNavigationProp，从@react-navigation/bottom-tabs 中导入 BottomTabNavigationProp 等等。

要注解这个 route，我们需要使用 @react-navigation/native 中的 RouteProp 类型:

import { RouteProp } from "@react-navigation/native";

type ProfileScreenRouteProp = RouteProp<RootStackParamList, "Profile">;

type Props = {
  route: ProfileScreenRouteProp,
};

这允许我们输入检查路由对象，例如 route.params。

总而言之:

import { RouteProp } from "@react-navigation/native";
import { StackNavigationProp } from "@react-navigation/stack";

type RootStackParamList = {
  Home: undefined,
  Profile: { userId: string },
  Feed: { sort: "latest" | "top" } | undefined,
};

type ProfileScreenRouteProp = RouteProp<RootStackParamList, "Profile">;

type ProfileScreenNavigationProp = StackNavigationProp<
  RootStackParamList,
  "Profile"
>;

type Props = {
  route: ProfileScreenRouteProp,
  navigation: ProfileScreenNavigationProp,
};

或者，你也可以导入一个泛型类型来定义相应导航器中的 navigation 和 route 的类型:

import { StackScreenProps } from "@react-navigation/stack";

type RootStackParamList = {
  Home: undefined,
  Profile: { userId: string },
  Feed: { sort: "latest" | "top" } | undefined,
};

type Props = StackScreenProps<RootStackParamList, "Profile">;

类似地，你可以从 @react-navigation/drawer 导入 DrawerScreenProps，从 @react-navigation/bottom-tabs 导入 BottomTabScreenProps 等。

然后可以使用 Props 类型来注解组件。

对于函数组件:

function ProfileScreen({ route, navigation }: Props) {
  // ...
}

对于 class 组件

class ProfileScreen extends React.Component<Props> {
  render() {
    // ...
  }
}

我们建议创建一个单独的 types.tsx 文件，保存类型并将它们导入到组件文件中，而不是在每个文件中重复它们。

# 嵌套导航器

嵌套导航器中的类型检查屏幕和参数

你可以通过为嵌套屏幕传递 screen 和 params 属性来在嵌套导航器中导航到屏幕:

navigation.navigate("Home", {
  screen: "Feed",
  params: { sort: "latest" },
});

为了能够检查它，我们需要从包含嵌套导航器的屏幕中提取参数。这可以使用 NavigatorScreenParams 工具来完成:

import { NavigatorScreenParams } from "@react-navigation/native";

type TabParamList = {
  Home: NavigatorScreenParams<StackParamList>,
  Profile: { userId: string },
};

组合导航的 props

当您嵌套导航器时，屏幕上的导航 prop 是多个导航 props 的组合。例如，如果我们在堆栈中有一个选项卡，navigation 将同时具有 jumpTo(来自选项卡导航器)和 push(来自堆栈导航器)。为了更容易地组合多个导航器的类型，你可以使用 CompositeNavigationProp 类型:

import { CompositeNavigationProp } from "@react-navigation/native";
import { BottomTabNavigationProp } from "@react-navigation/bottom-tabs";
import { StackNavigationProp } from "@react-navigation/stack";

type ProfileScreenNavigationProp = CompositeNavigationProp<
  BottomTabNavigationProp<TabParamList, "Profile">,
  StackNavigationProp<StackParamList>
>;

CompositeNavigationProp 类型有 2 个参数，第一个参数是主导航类型(拥有这个屏幕的导航器的类型，在我们的例子中是包含 Profile 屏幕的选项卡导航器)，第二个参数是辅助导航类型(父导航器的类型)。主导航类型应该始终将屏幕的路由名作为第二个参数。

对于多个父导航器，次要类型应该是嵌套的:

type ProfileScreenNavigationProp = CompositeNavigationProp<
  BottomTabNavigationProp<TabParamList, "Profile">,
  CompositeNavigationProp<
    StackNavigationProp<StackParamList>,
    DrawerNavigationProp<DrawerParamList>
  >
>;

# 注解 useNavigation

要注释我们从 useNavigation 中获得的 navigation，我们可以使用类型参数:

const navigation = useNavigation<ProfileScreenNavigationProp>();

需要注意的是，这并不是完全类型安全的，因为您使用的类型参数可能不正确，我们无法静态验证它。

# 注解 useRoute

要注释我们从 useRoute 中获得的 route，我们可以使用类型参数:

const route = useRoute<ProfileScreenRouteProp>();

需要注意的是，这并不是完全类型安全的，类似于 useNavigation。

# 注解 options 和 screenOptions

当您将选项传递给 Screen 或 screenOptions 给 Navigator 组件时，它们已经进行了类型检查，不需要做任何特殊操作。但是，有时您可能希望将选项提取到一个单独的对象，并且您可能希望对它进行注释。

为了注释选项，我们需要从导航器中导入相应的类型。例如，@react-navigation/stack 的 StackNavigationOptions:

import { StackNavigationOptions } from "@react-navigation/stack";

const options: StackNavigationOptions = {
  headerShown: false,
};

类似地，你可以从 @react-navigation/drawer 中导入 DrawerNavigationOptions，从@react-navigation/bottom-tabs 中导入 BottomTabNavigationOptions 等

# 注解 ref 和 NavigationContainer

当向 NavigationContainer 添加参考时，可以使用 NavigationContainerRef 类型对其进行注解。

import { NavigationContainerRef } from "@react-navigation/native";

// ...

const navigationRef = React.useRef < NavigationContainerRef > null;

使用 React.createRef 的例子

import { NavigationContainerRef } from '@react-navigation/native';

// ...

const navigationRef = React.createRef<NavigationContainerRef>();
