# 优化内存使用和性能

在 2.14.0 之前，每个平台上的所有屏幕本质上都是常规的原生视图，这将增加内存的使用，并使渲染树深入到一个沉重的应用程序中。这是你的应用程序比原生导航解决方案慢的原因之一。

随着 react-native-screens 的出现，通过引入原生导航组件(iOS 的 UIViewController, Android 的 FragmentActivity)，原生屏幕优化也可能出现在 React Navigation 中。通过使用 react-native-screens，每个本机平台都可以优化视图堆栈下屏幕的内存使用，并简化本机节点层次结构。您可以查看这里的对比，以了解性能的提高。

# 在使用 Expo 时设置

默认情况下，expo 已经包含了 react-native-screens，你所需要做的就是在你的导航栈被渲染之前(通常在 index.js 或 App.js 文件中)粘贴以下代码片段:

// Before rendering any navigation stack
import { enableScreens } from "react-native-screens";
enableScreens();
