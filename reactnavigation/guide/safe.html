# 支持安全区域

默认情况下，React Navigation 会确保导航器的元素在带有刘海屏的设备 (如 iPhone X) 和可能与应用内容重叠的 UI 元素上正确显示。这些项目包括:

  • 物理缺口
  • 状态栏覆盖
  • 在 iOS 上的 Home 活动指示器
  • Android 导航栏

没有这些项目重叠的地区称为 “安全区”。

我们尝试在导航器的 UI 元素上应用适当的插入，以避免与这些项重叠。其目标是(a)最大限度地利用屏幕(b)不隐藏内容，或通过物理显示切口或某些操作系统 UI 使其难以交互。

当 React Navigation 默认处理内置 UI 元素的安全区域时，你自己的内容也需要处理它，以确保内容不被这些项目隐藏。

我们很容易解决(a)将整个应用程序包装在一个带有填充的容器中，以确保所有内容不会被遮挡。但是这样做，我们浪费了屏幕上的大量空间，如下面的图片所示。我们想要的是右边的图像。

当 React Native 输出一个 SafeAreaView 组件时，它有一些固有的问题，例如，如果包含安全区域的屏幕正在动画，它会导致跳跃行为。此外，该组件只支持 iOS 10+，不支持旧的 iOS 版本或 Android。我们建议使用 react-native-safe-area-context (opens new window) 库以更可靠的方式处理安全区域。

本指南的其余部分提供了更多关于如何在 React 导航中支持安全区域的信息。

# 隐藏/自定义标题或标签栏

React Navigation 处理默认头部中的安全区域。但是，如果你使用的是自定义头文件，那么确保你的 UI 处于安全区域是很重要的。

例如，如果我不渲染头部或标签栏，什么也没有渲染

import * as React from "react";
import { Text, View } from "react-native";
import { NavigationContainer } from "@react-navigation/native";
import { createBottomTabNavigator } from "@react-navigation/bottom-tabs";
import { createStackNavigator } from "@react-navigation/stack";

function Demo() {
  return (
    <View
      style={{ flex: 1, justifyContent: "space-between", alignItems: "center" }}
    >
      <Text>This is top text.</Text>
      <Text>This is bottom text.</Text>
    </View>
  );
}
const Stack = createStackNavigator();
const Tab = createBottomTabNavigator();

export default function App() {
  return (
    <NavigationContainer>
      <Stack.Navigator initialRouteName="Home" headerMode="none">
        <Stack.Screen name="Home">
          {() => (
            <Tab.Navigator initialRouteName="Analytics" tabBar={() => null}>
              <Tab.Screen name="Analytics" component={Demo} />
              <Tab.Screen name="Profile" component={Demo} />
            </Tab.Navigator>
          )}
        </Stack.Screen>

        <Stack.Screen name="Settings" component={Demo} />
      </Stack.Navigator>
    </NavigationContainer>
  );
}

# 横屏模式

即使你使用默认的导航栏和标签栏-如果你的应用程序在横屏模式下工作，重要的是确保你的内容没有隐藏在传感器集群后面。

要解决这个问题，你可以再次在内容中应用安全区域插入。这不会与纵向模式下的导航栏或标签栏的默认行为冲突。

# 使用钩子来进行更多的控制

在某些情况下，您可能需要更多地控制应用哪些填充。例如，你只能通过改变 style 对象来应用顶部和底部的填充:

import { useSafeAreaInsets } from "react-native-safe-area-context";

function Demo() {
  const insets = useSafeAreaInsets();

  return (
    <View
      style={{
        paddingTop: insets.top,
        paddingBottom: insets.bottom,

        flex: 1,
        justifyContent: "space-between",
        alignItems: "center",
      }}
    >
      <Text>This is top text.</Text>
      <Text>This is bottom text.</Text>
    </View>
  );
}

类似地，您可以在 FlatList 的 contentContainerStyle 中应用这些填充，使内容避开安全区域，但在滚动时仍将它们显示在状态栏和导航栏下。

# 总结

  • 不要在 SafeAreaView 中包装你的整个应用程序，而是包装你屏幕中的内容
  • 使用 useSafeAreaInsets 钩子来控制插入的应用位置
