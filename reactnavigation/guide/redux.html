# Redux 集成

在 React Navigation 应用中使用 Redux 是非常容易的。这基本上和没有 React Navigation 没什么不同。

import { Provider } from "react-redux";
import { NavigationContainer } from "@react-navigation/native";

// Render the app container component with the provider around it
export default function App() {
  return (
    <Provider store={store}>
      <NavigationContainer>{/* Screen configuration */}</NavigationContainer>
    </Provider>
  );
}

请注意，我们将组件包装在 Provider 中，就像我们通常使用 react-redux 那样。是的! 现在你可以在整个应用中使用 connect。

# 在 options 中使用一个 connected 的组件

创建一个组件，将其 connect 到 store，然后在 title 中使用该组件。

function Counter({ value }) {
  return <Text>Count: {value}</Text>;
}

const CounterContainer = connect((state) => ({ value: state.count }))(Counter);

<Screen
  name="Test"
  component={TestScreen}
  options={{ title: () => <CounterContainer /> }}
/>

# 将您关心的状态作为参数传递给屏幕

如果该值预计不会更改，则可以将其作为参数从已连接的组件传递到另一个屏幕。

<Button
  title="Go to static counter screen"
  onPress={() =>
    props.navigation.navigate("StaticCounter", {
      count,
    })
  }
/>

function StaticCounter({ route }) {
  return (
    <View style={styles.container}>
      <Text style={styles.paragraph}>{route.params.count}</Text>
    </View>
  );
}

<RootStack.Screen
  name="StaticCounter"
  component={StaticCounter}
  options={({ route }) => ({ title: route.params.count })}
/>

# 我也可以在 Redux 中存储导航状态吗?

这是不可能的。我们不支持它，因为它太容易砸到自己的脚，减慢或破坏你的应用。

但是，可以使用 redux-devtools-extension (opens new window) 来检查导航状态和操作，也可以使用 devtools 包执行时间旅行调试。
