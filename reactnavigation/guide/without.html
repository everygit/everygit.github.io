# 在没有 navigation prop 的情况下进行导航

有时，您需要从无法访问 navigation prop 的地方（例如 Redux 中间件）触发导航操作。 在这种情况下，您可以从导航容器中分派导航操作。

如果您正在寻找一种从组件内部进行导航而无需向下传递 navigation prop 的方法，请参见 useNavigation。 当您可以访问 navigation prop 或使用 Navigation 时，请勿使用此方法，因为它的行为会有所不同，并且许多特定于屏幕的帮助程序方法将不可用。

您可以通过 ref 访问根导航对象，并将其传递给 RootNavigation，我们稍后将使用它进行导航。

// App.js

import { NavigationContainer } from "@react-navigation/native";
import { navigationRef } from "./RootNavigation";

export default function App() {
  return (
    <NavigationContainer ref={navigationRef}>{/* ... */}</NavigationContainer>
  );
}

在下一步中，我们定义 RootNavigation，这是一个简单的模块，具有分派用户定义的导航操作的功能。

// RootNavigation.js

import * as React from "react";

export const navigationRef = React.createRef();

export function navigate(name, params) {
  navigationRef.current?.navigate(name, params);
}

// add other navigation functions that you need and export them

然后，在您的任何 javascript 模块中，只需导入 RootNavigation 并调用从中导出的函数即可。 您可以在 React 组件之外使用这种方法，实际上，从它们内部使用它也一样有效。

// any js module
import * as RootNavigation from "./path/to/RootNavigation.js";

// ...

RootNavigation.navigate("ChatScreen", { userName: "Lucy" });

除了 navigate 之外，您还可以添加其他导航操作：

import { StackActions } from "@react-navigation/native";

export function push(...args) {
  navigationRef.current?.dispatch(StackActions.push(...args));
}

请注意，需要渲染堆栈导航器以处理此操作。 您可能需要检查文档是否有嵌套以了解更多详细信息 (opens new window)。

在编写测试时，您可以模拟导航功能，并断言是否使用正确的参数调用了正确的功能。

# 处理初始化

使用此模式时，您需要记住几件事，以避免应用程序崩溃。

  • ref 仅在导航容器渲染之后设置，这在处理深层链接时可以是异步的
  • 需要渲染导航器才能处理动作

如果您尝试在不渲染导航器的情况下进行导航，或者在导航器完成安装之前进行导航，则如果不处理，它将抛出您的应用程序并使其崩溃。因此，您需要添加其他检查来决定在应用挂载之前该怎么做。

例如，请考虑以下情形，您在应用程序中的某个位置有一个屏幕，该屏幕在 useEffect / componentDidMount 上调度了一个 redux 操作。您正在中间件中监听此操作，并在获得该操作时尝试执行导航。这将引发错误，因为此时父导航器尚未完成安装且尚未准备就绪。总是在子组件的 useEffect / componentDidMount 之后调用父组件的 useEffect / componentDidMount。

为避免这种情况，您可以设置一个引用来告诉您您的应用已完成安装，并在执行任何导航之前检查该引用。为此，我们可以在 NavigationContainer 中使用 onReady 回调：

// App.js

import { NavigationContainer } from "@react-navigation/native";
import { navigationRef, isReadyRef } from "./RootNavigation";

export default function App() {
  React.useEffect(() => {
    return () => {
      isReadyRef.current = false;
    };
  }, []);

  return (
    <NavigationContainer
      ref={navigationRef}
      onReady={() => {
        isReadyRef.current = true;
      }}
    >
      {/* ... */}
    </NavigationContainer>
  );
}

还要从我们的 RootNavigation 导出此引用：

// RootNavigation.js

import * as React from "react";

export const isReadyRef = React.createRef();

export const navigationRef = React.createRef();

export function navigate(name, params) {
  if (isReadyRef.current && navigationRef.current) {
    // Perform navigation if the app has mounted
    navigationRef.current.navigate(name, params);
  } else {
    // You can decide what to do if the app hasn't mounted
    // You can ignore this, or add these actions to a queue you can call later
  }
}

请注意，这仅在容器完成安装之前分派操作时处理这种情况。 如果不渲染任何导航器，仍然会出现错误。 必须渲染导航器才能分派动作。

如果不确定是否渲染了导航器，则可以调用 navigationRef.current.getRootState()，如果渲染了任何导航器，它将返回一个有效的状态对象，否则将返回undefined的状态对象。
