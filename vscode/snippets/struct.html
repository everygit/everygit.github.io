# 结构

代码片段有自己固定的结构。

# 示例

{
  "For Loop": {
    "prefix": ["for", "for-const"],
    "body": ["for (const ${2:element} of ${1:array}) {", "\t$0", "}"],
    "description": "A for loop."
  }
}
  • "For Loop" 是当前代码片段的名称，当没有提供 "description" 的时候，会在提示区域显示，同时会常态显示在列表的右侧。
  • "prefix" 提供一个或多个关键词用于触发智能感知。
  • "body" 提供一行或多行内容，在插入的时候会连接成多行，可以使用制表符进行内容的格式化。
  • "description" 可选，用来描述智能感知的内容。

另外，上述的示例代码中，存在着 3 个制表符 ${1:array} 、${2:element} 、$0，1 是最先的一个，然后按 tab 键，会到 2，0 永远都是在最后面的。这里的 array 和 element 都是占位符默认值，你可以不输入内容直接按 tab 键。

# 范围

你可以定义代码片段的范围，比如 typescript 文件 javascript 文件，又或者是 markdown 文件。甚至你可以指定项目。

# 单类型文件

在创建代码片段的时候，选择一种文件类型如 javascript.json 或 markdown.json 等等。

# 多类型文件

如果需要在一个文件中定义多种语言的代码片段，就不能使用单个类型的语言文件，我们就要创建一个通用的文件，一般情况下，文件名为 *.code-snippets.json。

那么，我们需要在配置中增加 scope 属性，来定义代码片段的范围，如果不定义 scope 属性，那么，范围就是全局的。

{
  "示例": {
    "prefix": "example",
    "scope": "typescript,typescriptreact,javascript,javascriptreact",
    "body": "hello world"
  }
}

# 单个项目

在创建代码片段文件的时候，会出现”新建 *** 文件夹的代码片段文件“。选择就好了，这样的话，代码片段只会作用于当前的文件夹。

在这里代码片段文件是定义全局的，如果需要的话，要使用 scope 来限定代码片段的作用范围。
