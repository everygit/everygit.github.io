# 在主应用中注册微应用

qiankun 与 single-spa 的区别就是入口文件， single-spa 需要的是 js 的路径，而 qiankun 是从 html 中自己解析 js 和 css

而 html 的路径，可以使用绝对的路径，而最后是通过 nginx 来做反向代理，使用相对的路径，这样可以避免域名更换造成的主程序需要更新的问题

如果使用 nginx 的话，只需要在 nginx 的代理处更换地址，而不需要修改代码

import { registerMicroApps, start } from "qiankun";

registerMicroApps([
  {
    name: "react app", // app name registered
    entry: "//localhost:7100",
    container: "#yourContainer",
    activeRule: "/yourActiveRule",
  },
  {
    name: "vue app",
    entry: { scripts: ["//localhost:7100/main.js"] },
    container: "#yourContainer2",
    activeRule: "/yourActiveRule2",
  },
]);

start();

原则上，主程序越简单越好 ε==(づ′▽`)づ

如果微应用不跟路由关联

import { loadMicroApp } from "qiankun";

loadMicroApp({
  name: "app",
  entry: "//localhost:7100",
  container: "#yourContainer",
});
