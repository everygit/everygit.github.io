# 主应用

主应用并不需要什么 react、vue 以及等等，只要是一个 webpack 创建的项目就行，当然了，如果不想用 webpack 也可以。

# 注册代码

假设我们这个是我们的入口 index.js，那么我们将在这里注册微应用，因为代码都是 bundle 好的，所以所谓的技术栈，在这里就毫不在意了。

另外，注册的地址，我们可以用下面的绝对地址，也可以用 nginx 来做反向代理，形成一个相对路径。

import { registerMicroApps, start } from "qiankun";

registerMicroApps([
  {
    name: "reactApp",
    entry: "//localhost:3000",
    container: "#container",
    activeRule: "/app-react",
  },
  {
    name: "vueApp",
    entry: "//localhost:8080",
    container: "#container",
    activeRule: "/app-vue",
  },
]);
// 启动 qiankun
start();

# HTML 模板

来看下上面配置的 container，其实与 react 中的 app 是一样的。

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>主应用的模板</title>
  </head>
  <body>
    <div id="container">
      <!-- 这里就是我们要用的容器 -->
    </div>
  </body>
</html>
